## 📅 오늘 배운 내용

### ✨ 1. 스프링 핵심 개념들 정리

1. **IoC(Inversion of Control, 제어의 역전)**
    - 객체 생성과 생명주기 관리를 개발자가 아닌 **스프링 컨테이너**가 담당함.
    - 개발자는 new로 객체를 생성하는 것이 아닌 스프링 컨테이너에 등록된 Bean을 사용함.
    - 핵심 키워드 -> ApplicationContext, Bean, @Component, @Configuration, @Bean
--- 
2. DI(Dependency Injection, 의존성 주입) 
    - IoC 구현하는 방식 중 하나
    - 객체가 사용할 다른 객체를 외부에서 주입받는 개념
    - 방법 : lombok의 @RequiredArgsConstructor 사용하고, **final** 멤버 변수 선언하면 자동으로 등록(생성자 주입), 생성자 주입, 세터 주입, 필드주입 
---
3. Bean과 스프링 컨테이너
    - Bean : 스프링이 관리하는 객체
    - 컨테이너 : Bean을 생성/초기화/소멸까지 관리
    - Bean 등록 방식 : 컴포넌트 스캔 -> @Component, @Service, @Repository, @Controller, 직접 등록 -> @Bean 메서드
---
4. AOP(Aspect Oriented Programming, 관점 지향 프로그래밍)
    - 공통 기능(로그,보안,트랜잭션 등)을 핵심 비즈니스 로직과 분리해서 관리하는 기법.
    - 예시 : 모든 서비스 메서드 실행 전에 로그 출력 -> AOP로 한 번에 적용 가능.
    - 관련 어노테이션 : @Aspect, @Around, @Before, @After
---
5. 트랜잭션 관리
   - DB 작업(삽입, 수정, 삭제)이 중간에 실패하면 전체 롤백 필요
   - 스프링은 @Transactional로 트랜잭션 편하게 관리 가능
---
6. Spring MVC
    - 웹 애플리케이션을 위한 스프링의 핵심 모듈
    - 동작 흐름 : DispatcherServlet -> Controller -> Service -> Repository
    - 핵심 어노테이션 : @Controller, @RestController, @RequestMapping, @GetMapping, @PostMapping ... 
    - DispatcherServlet 동작 흐름 
      -  톰캣 같은 서블릿 컨테이너가 요청 수신 -> DispatcherServlet으로 전달 -> 어떤 컨트롤러 메서드가 이 요청을 처리해야 하는지 찾음(URL,POST,GET 등) -> 찾은 컨트롤러 실행 전, 적절한 HandlerAdapter 찾고 컨트롤러 실행 준비
      -> @Controller나 @RestController에 있는 메서드 호출됨 -> 반환 타입이 뷰 이름일 경우 : 실제 뷰 파일 찾아서 렌더링, 반환 타입이 @ResponseBody/@RestController 라면 : ViewResolver대신 HttpMessageConverter가 동작하여 객체를 JSON 같은 형태로 변환 -> 렌더링 된 뷰나 JSON 데이터가 HTTP 응답으로 만들어지고, 톰캣이 응답을 클라이언트에 전달

8. JPA / Spring Data JPA
    - ORM(Object Relational Mappping)기술
    - 객체와 데이터베이스 매핑해서 SQL을 직접 쓰지 않고 DB접근 가능
    - @Entity, @Id, @GeneratedValue
    - Repository 인터페이스 -> findByName, findAll 등 메서드 자동 생성
    - 자세한 정리는 JPA 정리 부분 참고

9. Security 
    - 인증(Authentication), 인가(Authorization)를 지원하는 모듈
    - JWT, OAuth2 같은 인증 방식과 쉽게 통합
    - 핵심 클래스 : UserDetailService, AuthenticationManager
