# JPA는 왜 사용하는가?

## 1. 객체 지향과 관계형 DB 간의 간극 해소

-   자바는 **객체지향 언어**이고, RDB(Relational Database)는 **테이블
    기반**이라 구조가 다릅니다.\
-   예: 객체는 상속, 연관관계, 컬렉션 등을 쓰지만 DB는 테이블과 외래키,
    조인으로 표현해야 함.\
    ➡️ JPA는 이런 차이를 **자동으로 매핑**해 줍니다.

------------------------------------------------------------------------

## 2. SQL을 직접 쓰지 않고도 DB 조작 가능

-   기존 JDBC에서는 CRUD마다 **SQL을 직접 작성**해야 했습니다.\
-   JPA는 `EntityManager.persist(entity)` 같은 메서드 호출만으로 SQL을
    자동 생성/실행합니다.\
    ➡️ 코드가 단순해지고, **SQL 반복 작성 부담**이 줄어듭니다.

------------------------------------------------------------------------

## 3. 생산성 향상

-   개발자가 비즈니스 로직에만 집중할 수 있습니다.\
-   테이블 → 객체 변환, 쿼리 실행, 매핑 등 번거로운 작업을 JPA가 대신
    처리합니다.\
    ➡️ 코드량이 줄고 유지보수성이 좋아집니다.

------------------------------------------------------------------------

## 4. 데이터베이스 독립성

-   SQL은 DB마다 조금씩 차이가 있습니다(MySQL, Oracle, PostgreSQL...).\
-   JPA는 **표준 API**로 동작하기 때문에, DB를 바꿔도 코드 변경이
    최소화됩니다.\
    ➡️ `dialect`만 바꿔주면 대부분의 경우 애플리케이션은 그대로
    작동합니다.

------------------------------------------------------------------------

## 5. 성능 최적화 기능

-   1차 캐시, 지연 로딩(Lazy Loading), 벌크 연산, 변경 감지(Dirty
    Checking) 같은 기능 제공.\
    ➡️ 단순 SQL 자동화가 아니라, **최적화된 DB 접근 방식**을 제공합니다.

------------------------------------------------------------------------

## ✅ 정리

JPA는 **개발 생산성 + 유지보수성 + DB 독립성 + 성능 최적화** 때문에
사용합니다.

> 단, 단순 프로젝트라면 오히려 **JDBC + MyBatis**가 직관적일 수 있고,\
> JPA는 러닝커브가 있다는 단점도 있습니다.
