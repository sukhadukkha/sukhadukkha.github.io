# Docker를 공부하고 싶어진 이유 + DevOps까지 !

## 개요
저는 이번 3학년 2학기에 앱을 개발하는 수업의 팀장을 맡아 처음 팀 프로젝트를 수행하게 되었습니다. <br>
저를 포함한 팀원들 모두 팀프로젝트 경험은 없었고, 다들 사용해본 프레임워크나 자신의 주 사용 언어가 없는 상황이었습니다. <br>
그나마 저는 방학에 Spring boot를 어느정도 공부했었기 때문에 프로젝트를 주도하며 팀장을 맡게 되었고 서버는 Spring boot, 프론트와 앱은 Kotlin으로 개발을 진행하게 되었습니다. <br>
저는 백엔드 개발의 모든 부분을 진행했고, API 명세를 팀원들에게 공유해주고, UI,UX는 팀원들에게 맡기고 진행하려는데 문제가 발생했습니다. <br>

## 발생한 문제점

- `Java가 안깔려 있는 팀원`
  - Java가 안깔려있는 팀원이 있었고, 깔려있어도 각기 다른 버전을 가지고 있어서 프로젝트 빌드가 안되는 상황이 발생했습니다.
  - Docker라는 도구를 모르고 있었기 때문에 각자 제가 사용한 자바 버전을 설치하여 진행해야 했습니다.
- `백엔드 개발에 사용한 Local DB, 다양한 secret key의 문제로 인한 서버 제공 문제`
  - 저는 저의 Local DB 설정, 다양한 secret key들을 application.properties에 설정해 두고 사용했습니다.
  - **이는 저만 서버에 접근할 수 있고, 팀원들이 프로젝트의 backend Repo를 clone하여도 번거롭게 자신의 Local설정을 해야만 각각 본인의 Local에서 서버를 실행 할 수 있었습니다.**
  - 팀원들에게 이런 복잡한 상황을 해결해줄 방법을 부족한 저는 당장 제시해주지 못했고 프로젝트는 계속 진행되었습니다.
  - **팀원들이 프론트 코드를 테스트해보려면 항상 제가 ngrok를 통해 서버를 제공해줘야 했고, 개발 과정에서 굉장한 불편함을 느끼게 되었습니다.**
  - 이 문제를 어떻게 해결해야 하나 생각해보다가, Docker라는 도구를 알게되었고, 학기가 끝나고 바로 공부 해봐야겠다는 생각이 들었습니다. (프로젝트가 거의 끝나가서 바로 도입해보기에는 무리라고 판단)

# 더 나아가 Cloud + DevOps까지

이러한 상황을 겪으면서, 배포까지 고려하는 팀프로젝트를 진행할 때 필히 **개발의 전 과정을 팀원들이 통합된 환경에서 매끄럽게 진행하게 할 수 있는 인원이 필요하다고 생각이 들었습니다.**<br>
이 경험을 통해 단순히 백엔드 개발을 잘 하는 것도 중요하지만, 주니어 개발자들도 AI Agent의 도움으로 코드 작성의 효율성은 굉장히 많이 증가되어 있는 현재의 상황과 넘처나는 자료들을 통해 무엇이든 학습할 수 있는 현재의 상황에서 **프로젝트의 인프라 전체를 설계하는 능력또한 중요해지고 있다는 생각을 하게 되었습니다.** <br>
저는 그 능력을 향상시키고, 4학년 1학기에 진행 할 캡스톤 프로젝트에서 `팀원들에게 복잡한 인프라 설정 외의 개발에만 집중할 수 있는 환경을 제공해주고, 빌드, 테스트, 배포까지 매끄럽게 진행해보기 위해` Docker, AWS, Linux, github Actions를 통한 CI/CD 환경 구축을 공부해야겠다는 생각을 가지게 되었습니다.<br>
캡스톤 프로젝트에서의 Spring boot를 통한 백엔드 개발 및 인프라 구축을 맡은 저는 이번 겨울 방학에 `구체적으로 Docker, Linux (RHCSA), AWS(AWS SAA), github Actions 공부를 병행하며 DevOps 역량을 키워나갈 계획입니다.` <br>
