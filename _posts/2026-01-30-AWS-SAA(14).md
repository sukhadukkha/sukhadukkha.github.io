## Amazon Athena

- 정의
    - Amazon S3에 저장된 데이터를 SQL로 직접 조회할 수 있는 서버리스 분석 서비스
    - 인프라 관리 필요 없음
    - 쿼리한 만큼만 비용 지불

- 주요 특징
    - 데이터는 S3에 그대로 저장
    - 표준 SQL 사용
    - 대용량 데이터(TB~PB) 분석 가능
    - Ad-hoc 분석에 적합

- 지원 데이터 형식
    - CSV
    - JSON
    - Parquet
    - ORC
    - Avro
    - Parquet / ORC 사용 시 성능 향상 및 비용 절감

- 스키마 관리
    - AWS Glue Data Catalog 사용
    - 테이블 메타데이터만 저장
    - 실제 데이터는 S3에 존재

- 비용 모델 (중요)
    - 스캔한 데이터 양 기준 과금
    - 비용 최적화 방법
        - 컬럼 기반 포맷(Parquet, ORC) 사용
        - 파티셔닝 적용
        - 필요한 컬럼만 조회

- 사용 사례
    - 적합한 경우
        - 로그 분석
        - 데이터 레이크 분석
        - 간헐적인 분석 쿼리
        - 서버리스 분석 환경 필요
    - 부적합한 경우
        - OLTP 트랜잭션 처리
        - 빈번한 데이터 업데이트
        - 초저지연 응답 요구

- 비교
    - Athena vs Redshift
        - Athena
            - 서버리스
            - S3 데이터 직접 조회
            - 쿼리당 과금
            - Ad-hoc 분석
        - Redshift
            - 클러스터 기반
            - 내부 스토리지
            - 노드 시간 과금
            - 대규모 정형 분석

- 연계 서비스
    - Amazon QuickSight
        - Athena 쿼리 결과 시각화
        - 데이터 레이크 보고 솔루션 구성 가능

- 시험용 한 줄 요약
    - Athena는 S3 데이터를 서버리스 환경에서 SQL로 분석하는 서비스


## Amazon Redshift 

- **무엇인가?**
    - AWS에서 제공하는 **관리형 데이터 웨어하우스**
    - 정형 데이터(OLAP) 분석에 최적화
    - SQL로 쿼리 가능, PB급 데이터 처리 가능

- **특징**
    - 클러스터 기반: 여러 노드로 구성
    - 컬럼 기반 저장 → 필요한 컬럼만 읽어 속도 향상
    - MPP(병렬 처리) → 대규모 데이터 빠르게 처리
    - 자동 백업, 스냅샷 제공
    - **Redshift Spectrum**: S3 데이터도 SQL로 조회 가능

- **데이터 로딩**
    - COPY 명령으로 S3, DynamoDB 등에서 로드
    - 압축 가능 → 성능 & 비용 최적화

- **성능 최적화**
    - SORT KEY, DIST KEY 설정 → 쿼리 속도 개선
    - 자동 Vacuum & Analyze → 테이블 최적화

- **시험용 비교**
    - **Redshift**
        - 클러스터 필요
        - 내부 저장 데이터 분석
        - 반복적/정형 분석에 강함
        - 과금: 노드 시간 기준
    - **Athena**
        - 서버리스
        - S3 데이터 직접 조회
        - Ad-hoc 분석에 강함
        - 과금: 쿼리 스캔량 기준

- **연계 서비스**
    - S3 → 데이터 로드, Spectrum 사용
    - QuickSight → 시각화
    - Glue → 데이터 카탈로그 연동

- **시험용 한 줄 요약**
    - **Redshift = 대규모 정형 데이터 분석용 관리형 데이터 웨어하우스**

## Amazon OpenSearch 

- **무엇인가?**
    - AWS에서 제공하는 **검색 및 로그 분석 서비스**
    - Elasticsearch 기반, 완전 관리형
    - 로그, 메트릭, 텍스트 데이터 검색/분석에 최적화

- **주요 특징**
    - 완전 관리형: 인프라 관리 필요 없음
    - 실시간 검색 및 분석 가능
    - Kibana/ OpenSearch Dashboards와 통합 → 시각화 가능
    - 스케일링: 노드 추가/삭제 가능
    - AWS 보안 통합: IAM, VPC, KMS

- **사용 사례**
    - 로그 분석 (CloudWatch, Application logs)
    - 실시간 텍스트 검색 (웹사이트, 앱)
    - 모니터링 대시보드 구축
    - 보안 이벤트 검색 및 분석

- **시험 포인트**
    - OpenSearch = Elasticsearch 관리형 서비스
    - 로그 수집과 실시간 분석에 적합
    - S3 + Lambda + OpenSearch 연동 가능
    - Athena/Redshift와 달리 **검색과 로그 분석** 중심

- **연계 서비스**
    - CloudWatch Logs → 실시간 로그 수집
    - Kinesis Firehose → 로그 스트리밍
    - S3 → 데이터 아카이빙 및 분석 연계
    - QuickSight → 시각화 가능

- **시험용 한 줄 요약**
    - **OpenSearch = 로그/텍스트 데이터 검색 및 분석용 AWS 관리형 서비스**


## Amazon EMR 

- **무엇인가?**
    - AWS에서 제공하는 **빅데이터 처리/분석 서비스**
    - Apache Hadoop, Spark, Hive, Presto 등을 관리형으로 제공
    - 대규모 데이터 처리와 분석에 최적화

- **주요 특징**
    - 완전 관리형 클러스터: 노드 프로비저닝, 소프트웨어 설치 자동화
    - Auto Scaling 지원 → 처리량에 따라 노드 자동 추가/제거 가능
    - S3, DynamoDB, RDS 등 다양한 데이터 소스와 연계
    - 비용 최적화 가능: Spot Instance 지원

- **사용 사례**
    - 대규모 데이터 처리 (ETL, 로그 분석, 데이터 정제)
    - 데이터 웨어하우스 로딩 전 처리
    - 기계 학습 전 데이터 전처리
    - 대규모 SQL 분석(Hive, Presto 사용)

- **시험 포인트**
    - EMR = **관리형 Hadoop/Spark 클러스터**
    - S3 데이터를 처리하고 분석하는 경우 자주 사용
    - Redshift, Athena와 비교:
        - Redshift: 정형 데이터, OLAP, SQL 분석
        - Athena: 서버리스, ad-hoc SQL 분석
        - EMR: 비정형/반정형 데이터, 대규모 배치 처리, 복잡한 데이터 파이프라인

- **연계 서비스**
    - S3 → 원시 데이터 저장/읽기
    - Lambda/Kinesis → 실시간 스트리밍 처리 가능
    - Glue → ETL 파이프라인 구축
    - Redshift → 데이터 웨어하우스로 로딩

- **시험용 한 줄 요약**
    - **EMR = 대규모 데이터 처리와 분석을 위한 관리형 Hadoop/Spark 클러스터**


## Amazon QuickSight 

- **무엇인가?**
    - AWS에서 제공하는 **클라우드 기반 BI(Business Intelligence) 서비스**
    - 데이터 시각화 및 대시보드 생성에 최적화
    - 서버리스 구조 → 인프라 관리 불필요

- **주요 특징**
    - 다양한 데이터 소스 연결 가능
        - S3, Redshift, Athena, RDS, DynamoDB 등
    - 대시보드 공유 및 권한 설정 가능(User,Group)
    - ML Insights 제공 → 자동 패턴, 예측 분석
    - SPICE 인메모리 엔진 → 빠른 쿼리 성능
    - 구독/이메일 보고서 기능 제공

- **사용 사례**
    - 데이터 시각화 및 보고서 제공
    - 경영진/팀별 KPI 대시보드
    - 데이터 탐색 및 분석

- **시험 포인트**
    - QuickSight = **AWS BI/시각화 서비스**
    - SPICE 엔진 → 빠른 분석, 서버리스 구조
    - 다양한 AWS 데이터 소스와 통합 가능
    - 사용자별 접근 권한 관리 가능

- **연계 서비스**
    - S3 → 데이터 소스
    - Athena → 서버리스 쿼리
    - Redshift → 데이터 웨어하우스
    - EMR → 대규모 데이터 분석 결과 시각화

- **시험용 한 줄 요약**
    - **QuickSight = AWS 클라우드 기반 BI/시각화 서비스**


## AWS Glue

- **무엇인가?**
    - AWS에서 제공하는 **서버리스 ETL(Data Extract, Transform, Load) 서비스**
    - 데이터 수집, 정제, 변환, 로딩을 자동화
    - 데이터 웨어하우스/데이터 레이크와 통합 최적화

- **주요 특징**
    - 서버리스 → 인프라 관리 불필요
    - **ETL 작업 자동 생성 가능** (데이터 스키마 분석 후 스크립트 생성)
    - 데이터 카탈로그(Data Catalog) 제공 → 메타데이터 관리
    - Python/Scala 기반 Spark ETL 지원
    - 크롤러(Crawler) → S3, RDS, JDBC 데이터 스키마 자동 추출

- **사용 사례**
    - 데이터 레이크 구축: S3 데이터 정제 후 Redshift 로딩
    - ETL 파이프라인 자동화
    - 다양한 데이터 소스 통합
    - Athena, Redshift, QuickSight와 연동

- **시험 포인트**
    - Glue = **서버리스 ETL + 데이터 카탈로그**
    - 데이터 레이크에서 데이터를 준비/정리하는 서비스
    - 크롤러 → 메타데이터 자동 생성
    - Job → 변환/정제/로딩 작업 실행
    - Glue Data Catalog → Athena, Redshift, QuickSight에서 사용 가능

- **연계 서비스**
    - S3 → 데이터 소스 및 데이터 레이크
    - Redshift → 데이터 웨어하우스
    - Athena → 서버리스 쿼리
    - QuickSight → 시각화
    - Lambda/Kinesis → 실시간 데이터 파이프라인 연동 가능

- **시험용 한 줄 요약**
    - **AWS Glue = 서버리스 ETL 서비스 + 데이터 카탈로그로 데이터 레이크/웨어하우스 준비**


## AWS Data Lake 

- **무엇인가?**
    - 조직의 모든 데이터를 **중앙에 모아 저장**하는 저장소
    - AWS에서는 **S3**를 기반으로 구축
    - 정형, 반정형, 비정형 데이터 모두 저장 가능
    - 분석, BI, ML 등 다양한 서비스와 연계 가능

- **주요 특징**
    - 스케일 무제한 → PB급 데이터 처리 가능
    - 비용 효율적 → S3 스토리지 계층(Standard, IA, Glacier) 활용
    - 메타데이터 관리 → **AWS Glue Data Catalog**
    - 데이터 접근 제어 → IAM, Lake Formation, S3 정책

- **권한 관리 (Lake Formation)**
    - 중앙에서 데이터 접근 권한 통합 관리 가능
    - 사용자(User)와 그룹(Group) 단위 접근 제어
    - 테이블/컬럼/행(row) 수준 접근 제어 가능
    - 분석 서비스(Athena, Redshift Spectrum, EMR, QuickSight)에 **일괄 적용**
    - 신규 사용자/그룹 추가 시 중앙 정책만 업데이트하면 자동 반영

- **사용 사례**
    - 다양한 소스 데이터 통합 저장
    - Athena, Redshift Spectrum, EMR에서 분석
    - ML/AI 학습 데이터 준비
    - 실시간 스트리밍 데이터 수집 후 분석

- **시험용 한 줄 요약**
    - **AWS 데이터 레이크 = 중앙에 모든 데이터를 모아 분석 가능한 저장소 + Lake Formation으로 중앙 권한 관리**


## Amazon MSK 

- **무엇인가?**
    - AWS에서 제공하는 **관리형 Apache Kafka 서비스**
    - 실시간 스트리밍 데이터 처리용
    - Kafka 클러스터 운영/관리를 AWS가 대신 수행

- **주요 특징**
    - Apache Kafka와 완전 호환
    - 고가용성: 여러 AZ에 브로커 분산
    - 자동 복구 및 패치 관리
    - IAM, VPC, 보안 그룹, KMS 연동
    - 대규모 실시간 데이터 스트리밍 처리 가능

- **언제 쓰나?**
    - 실시간 이벤트 스트리밍
    - 로그, 클릭 스트림 수집
    - 마이크로서비스 간 비동기 통신
    - 데이터 레이크로 실시간 데이터 적재

- **시험 포인트**
    - MSK = **관리형 Kafka**
    - 대규모 실시간 스트리밍 데이터 처리
    - Kinesis보다 **오픈소스 Kafka 호환**이 필요할 때 사용
    - 실시간 파이프라인 구축 시 자주 등장

- **데이터 레이크 연계**
    - MSK → Kinesis Firehose / Kafka Connect
    - S3에 실시간 데이터 적재
    - Athena, EMR, OpenSearch에서 분석 가능

- **비교 (시험 자주 나옴)**
    - MSK vs Kinesis
        - MSK
            - Apache Kafka 호환
            - 클러스터 관리형
            - Kafka 생태계 활용 가능
        - Kinesis
            - AWS 네이티브
            - 완전 서버리스
            - AWS 서비스와 빠른 통합

- **시험용 한 줄 요약**
    - **MSK = Apache Kafka를 AWS에서 관리형으로 제공하는 실시간 스트리밍 서비스**
