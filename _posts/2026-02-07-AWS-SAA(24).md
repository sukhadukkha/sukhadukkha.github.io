## AWS Well-Architected Tool 간단 정리

---

## 1. Well-Architected Tool 개요

- AWS 워크로드가 **모범 사례(Best Practice)**에 맞는지 평가하는 도구
- 아키텍처를 **체계적으로 점검**하고 개선점 도출

---

## 2. 평가 기준 (6대 기둥)

1. 운영 우수성 (Operational Excellence)
2. 보안 (Security)
3. 안정성 (Reliability)
4. 성능 효율성 (Performance Efficiency)
5. 비용 최적화 (Cost Optimization)
6. 지속 가능성 (Sustainability)

---

## 3. 주요 기능

- 질문 기반 워크로드 평가
- 리스크 항목 식별
- 개선 권장 사항 제공
- 점수화 및 리포트 제공

---

## 4. 언제 사용하는가 (시험 포인트)

- 시스템 설계 검토
- 운영 중인 아키텍처 점검
- 비용/보안/안정성 개선 필요 시

→ **Well-Architected Tool**

---

## 5. 주의 사항

- 리소스를 자동으로 수정 ❌
- 진단 및 가이드 제공 도구
- 실제 개선 작업은 사용자가 수행

---

## 6. 한 줄 요약

- **Well-Architected Tool = AWS 모범 사례 기준으로 아키텍처를 점검하는 평가 도구**

---

## AWS Trusted Advisor 간단 정리

---

## 1. Trusted Advisor 개요

- AWS 계정을 **모범 사례 기준으로 자동 점검**하는 서비스
- 비용, 보안, 성능, 가용성 등을 지속적으로 검사

---

## 2. 점검 영역 (시험 중요)

1. 비용 최적화 (Cost Optimization)
2. 보안 (Security)
3. 성능 (Performance)
4. 내결함성 (Fault Tolerance)
5. 서비스 한도 (Service Limits)

---

## 3. 주요 기능

- 계정 리소스 상태 자동 분석
- 위험 요소 알림
- 개선 권장 사항 제공
- 일부 항목은 자동 새로고침 가능

---

## 4. 무료 vs 유료 (시험 포인트)

### 무료 (Basic / Developer Support)
- 보안 핵심 항목 일부
- 서비스 한도 일부

### 유료 (Business / Enterprise Support)
- 모든 체크 항목
- 실시간 모니터링
- CloudWatch 연동

---

## 5. 언제 사용하는가 (시험 시나리오)

- 비용 낭비 리소스 탐지
- 보안 설정 오류 확인
- 계정 전반 상태 점검

→ **Trusted Advisor**

---

## 6. Well-Architected Tool과 차이

- Trusted Advisor: **계정 리소스 자동 점검**
- Well-Architected Tool: **워크로드 설계 평가**

---

## 7. 한 줄 요약

- **Trusted Advisor = AWS 계정을 자동으로 점검해주는 건강검진 도구**

---

## AWS Rekognition 간단 정리

---

## 1. Rekognition 개요

- 이미지와 비디오를 분석하는 **AI 기반 컴퓨터 비전 서비스**
- 별도 모델 학습 없이 바로 사용 가능

---

## 2. 주요 기능 (시험 중요)

- 객체(Object) 및 장면(Scene) 인식
- 얼굴 감지 및 얼굴 비교
- 유명 인물(Celebrity) 인식
- 텍스트 인식 (OCR)
- 부적절한 콘텐츠(성인/폭력) 감지

---

## 3. 입력 데이터

- 이미지 (JPEG, PNG)
- 비디오 (S3에 저장된 파일)

---

## 4. 대표 사용 사례

- 사용자 얼굴 인증
- 사진 자동 태깅
- 동영상 콘텐츠 필터링
- 신원 확인 시스템

---

## 5. 시험 시나리오 포인트

- 이미지/비디오 분석 필요
- 얼굴/객체/텍스트 인식 요구
- ML 모델 직접 학습 ❌

→ **Rekognition**

---

## 6. 주의 사항 (시험)

- 얼굴 **인식(Identification)** ≠ 얼굴 **인증(Authentication)**
- 데이터는 S3 기반으로 처리
- 실시간 스트리밍은 Kinesis Video Streams 연계

---

## 7. 한 줄 요약

- **Rekognition = 이미지·비디오를 자동 분석하는 AWS AI 서비스**

## AWS Transcribe 간단 정리

---

## 1. Transcribe 개요

- 음성을 **텍스트로 변환(Speech-to-Text)** 하는 서비스
- 별도 모델 학습 없이 바로 사용 가능

---

## 2. 주요 기능 (시험 중요)

- 실시간(Streaming) 음성 변환
- 배치(Batch) 음성 변환
- 화자 구분 (Speaker Identification)
- 타임스탬프 제공
- 자동 언어 감지

---

## 3. 입력 / 출력

- 입력: 음성 파일 (WAV, MP3 등) 또는 실시간 오디오 스트림
- 출력: 텍스트 (JSON, SRT 등)

---

## 4. 대표 사용 사례

- 회의 녹취 자동 생성
- 콜센터 통화 기록
- 자막 생성
- 음성 검색 시스템

---

## 5. 시험 시나리오 포인트

- 음성을 텍스트로 변환해야 한다
- 실시간 또는 비동기 변환 필요
- ML 모델 직접 학습 ❌

→ **Transcribe**

---

## 6. 유사 서비스 비교 (시험 함정)

- **Transcribe**: 음성 → 텍스트
- **Polly**: 텍스트 → 음성
- **Comprehend**: 텍스트 의미 분석

---

## 7. 한 줄 요약

- **Transcribe = 음성을 자동으로 텍스트로 변환하는 AWS 서비스**

- 고품질 음성 합성
- 다양한 언어 / 성별 / 억양 지원
- SSML(Speech Synthesis Markup Language) 지원
- 실시간 및 배치 음성 생성

---

## 3. 입력 / 출력

- 입력: 텍스트 또는 SSML
- 출력: 음성 파일 (MP3, WAV 등)

---

## 4. 대표 사용 사례

- 안내 음성 시스템
- 오디오북 생성
- 챗봇 음성 응답
- 접근성(Accessibility) 기능

---

## 5. 시험 시나리오 포인트

- 텍스트를 음성으로 변환해야 한다
- 다국어 음성 지원 필요
- ML 모델 직접 학습 ❌

→ **Polly**

---

## 6. 유사 서비스 비교 (시험 함정)

- **Polly**: 텍스트 → 음성
- **Transcribe**: 음성 → 텍스트
- **Lex**: 음성/텍스트 기반 챗봇

---

## 7. 한 줄 요약

- **Polly = 텍스트를 자연스러운 음성으로 변환하는 AWS 서비스**

## AWS Lex + Amazon Connect 간단 정리

---

## 1. Lex 개요

- 음성 및 텍스트 기반 **챗봇 서비스**
- 자연어 이해(NLU) 기반으로 사용자 의도(Intent) 파악

---

## 2. Lex 주요 기능 (시험 중요)

- 음성 / 텍스트 입력 처리
- Intent, Slot 기반 대화 설계
- 다국어 지원
- Lambda 연동으로 비즈니스 로직 처리

---

## 3. Amazon Connect 개요

- AWS 기반 **클라우드 콜센터 서비스**
- 인프라 관리 없이 콜센터 구축 가능

---

## 4. Amazon Connect 주요 기능

- 음성/채팅 고객 응대
- 자동 콜 분배 (ACD)
- 콜 플로우 시각적 설계
- 실시간 모니터링 및 통계

---

## 5. Lex + Connect 연동 구조 (시험 포인트)

1. 고객 음성 입력
2. Amazon Connect 수신
3. Lex가 음성 인식 + 의도 분석
4. Lambda로 처리
5. 응답 반환 (음성/텍스트)

---

## 6. 대표 사용 사례

- 자동 고객 상담 (IVR)
- 주문/예약 시스템
- FAQ 자동 응답
- 콜센터 자동화

전화 수신: 고객이 Amazon Connect 번호로 전화를 건다.<br>

Lex 트리거: Connect의 고객 흐름(Contact Flow) 내에서 Amazon Lex 봇을 호출한다.<br>

의도 파악: Lex가 고객의 음성을 듣고 "비밀번호를 초기화하고 싶어요"라는 의도(Intent)를 파악한다.<br>

Lambda 실행 (옵션): Lex가 백엔드 로직인 AWS Lambda를 실행하여 DB(RDS, DynamoDB)에서 사용자 정보를 확인하거나 비밀번호를 바꾼다.<br>

응답 전달: 처리 결과나 다음 안내 사항을 Lex가 다시 음성으로 고객에게 전달한다.<br>

상담원 연결: 만약 AI가 해결할 수 없는 복잡한 문제라면 Connect가 즉시 상담원에게 통화를 넘긴다.<br>

---

## 7. 시험 시나리오 구분법

- **대화형 챗봇** → Lex
- **클라우드 콜센터** → Connect
- **자동 음성 상담** → Lex + Connect

---

## 8. 한 줄 요약

- **Lex = 대화형 챗봇**
- **Connect = 클라우드 콜센터**
- **Lex + Connect = AI 기반 자동 고객 응대**

## AWS Comprehend Medical 간단 정리

---

## 1. Comprehend Medical 개요

- 의료 텍스트에서 **의학적 정보 추출**에 특화된 NLP 서비스
- 임상 기록, 진단서, 처방전 분석에 사용

---

## 2. 주요 기능 (시험 중요)

- 의료 개체 인식  
  (질병, 증상, 약물, 해부학적 부위 등)
- 의료 용어 관계 분석
- ICD-10-CM, RxNorm 등 의료 코드 매핑
- PHI(개인 건강 정보) 감지

---

## 3. 입력 / 출력

- 입력: 의료 관련 텍스트
- 출력: 구조화된 의료 정보 (JSON)

---

## 4. 대표 사용 사례

- 전자의무기록(EMR) 분석
- 임상 문서 자동 분류
- 의료 데이터 구조화
- 의료 연구 및 분석

---

## 5. 시험 시나리오 포인트

- **의료/헬스케어 텍스트 분석**
- 일반 NLP가 아닌 의료 특화 분석
- ML 모델 직접 학습 ❌

→ **Comprehend Medical**

---

## 6. Comprehend vs Comprehend Medical

- Comprehend: 일반 텍스트 분석
- Comprehend Medical: 의료 텍스트 분석 전용

---

## 7. 한 줄 요약

## AWS SageMaker 간단 정리

---

## 1. SageMaker 개요

- 머신러닝 모델을 **구축·학습·배포**까지 한 번에 지원하는 서비스

- 자동 모델 튜닝 (AutoML)
- 내장 알고리즘 및 커스텀 모델 지원
- 노트북 기반 개발 환경 제공

## AWS Kendra 간단 정리

---

## 1. Kendra 개요

- 문서에서 정보를 찾아주는 **AI 기반 엔터프라이즈 검색 서비스**
- 자연어 질문에 대해 **의미 기반 검색 결과** 제공

---

## 2. 주요 기능 (시험 중요)

- 자연어 질의 처리
- 문서 의미 분석 (Semantic Search)
- 다양한 데이터 소스 커넥터 지원
- 접근 제어(ACL) 기반 검색 결과 제한

---

## 3. 입력 / 출력

- 입력: 문서 데이터  
  (S3, SharePoint, RDS, 웹사이트 등)
- 출력: 관련 문서 및 답변

---

## 4. 대표 사용 사례

- 사내 문서 검색 시스템
- 고객 지원 지식 베이스
- FAQ 자동 검색
- 기업용 포털 검색

데이터 연결: 인덱싱할 데이터 소스(예: 네가 공부한 자료가 담긴 S3 버킷)를 Kendra와 연결한다.<br>

인덱싱: Kendra가 문서를 읽고 의미를 분석하여 검색 가능한 형태로 저장한다.<br>

검색 실행: 사용자가 웹 인터페이스나 API를 통해 질문을 던진다.<br>

결과 반환: 단순히 문서 목록만 주는 것이 아니라, 문서 내의 구체적인 답변과 관련 정보를 함께 제공한다.<br>

---

## 5. 시험 시나리오 포인트

- **대량 문서 검색**
- 키워드가 아닌 의미 기반 검색
- ML 모델 직접 학습 ❌

→ **Kendra**

## 6. 유사 서비스 비교 (시험 함정)

- **Kendra**: 엔터프라이즈 문서 검색
- **OpenSearch**: 로그/텍스트 검색 엔진
- **Comprehend**: 텍스트 의미 분석

---

## 7. 한 줄 요약

- **Kendra = 문서를 이해하고 답을 찾아주는 AI 검색 서비스**

## AWS Textract 간단 정리

---

## 1. Textract 개요

- 스캔 문서나 이미지에서 **텍스트와 구조화된 데이터**를 추출하는 서비스
- OCR을 넘어 **표(Table)와 폼(Form) 인식** 지원

---

## 2. 주요 기능 (시험 중요)

- 인쇄 텍스트 및 손글씨 인식
- 테이블(Table) 데이터 추출
- 폼(Form) 키-값(Key-Value) 추출
- 비동기 대량 문서 처리

---

## 3. 입력 / 출력

- 입력: 문서 이미지 또는 PDF  
  (JPEG, PNG, PDF 등)
- 출력: 추출된 텍스트 및 구조화 데이터 (JSON)

---

## 4. 대표 사용 사례

- 계약서/청구서 자동 처리
- 영수증 및 세금 문서 분석
- 문서 디지털화
- 금융/보험 서류 자동화

---

## 5. 시험 시나리오 포인트

- **문서 이미지에서 표/폼 데이터 추출**
- 단순 OCR 이상의 구조 인식 필요
- ML 모델 직접 학습 ❌

→ **Textract**

---

## 6. 유사 서비스 비교 (시험 함정)

- **Textract**: 문서 구조 인식
- **Rekognition**: 이미지/비디오 분석
- **Comprehend**: 텍스트 의미 분석

---

## 7. 한 줄 요약

- **Textract = 문서 이미지에서 구조화된 데이터를 추출하는 OCR 서비스**
