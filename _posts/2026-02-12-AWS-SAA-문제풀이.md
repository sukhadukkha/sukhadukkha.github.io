## IAM (Identity and Access Management)

- IAM
  - User
  - Role
  - Group
  - Policy
- **Identity-based policy**
  - User/Role/Group 에 붙음
  - ex) User -> S3 읽기 가능
  - Identity policy
- Resource-based policy
  - S3 / Lambda / SQS ... 
  - Resource policy
  - ex) 계정 A의 S3 ->  계정B가 S3에 접근하는 것을 허용
- ex) 계정 A의 S3에 계정 B가 접근 해야 한다?
  - 요청자 권한(Identity) 허용 + 리소스 허용 둘 다 필요
  - 계정 B 사용자가 계정 A 버킷 읽는 policy
  - 계정 A의 버킷 정책
```json
{
 "Version": "2012-10-17",
 "Statement": [
  {
   "Effect": "Allow",
   "Action": [
     "s3:GetObject"
   ],
   "Resource": "arn:aws:s3:::mybucket/*"
  }
 ]
}
```

```json
{
 "Effect": "Allow",
 "Principal": {
   "AWS": "arn:aws:iam::ACCOUNT_B_ID:root"
 },
 "Action": "s3:GetObject",
 "Resource": "arn:aws:s3:::mybucket/*"
}
```


## AWS Beanstalk 

- 코드만 올리면 AWS가 서버, 배포, 스케일링 자동 관리해주는 서비스(관리형 Paas)


## AWS Organization

- 여러 AWS 계정을 중앙에서 관리하는 서비스
- 계정 그룹화, 정책 적용, 통합 결제 등을 제공한다.
- 조직은 하나의 관리 계정과 여러 멤버 계정으로 구성된다.
- **각 AWS 계정은 동시에 하나의 조직에만 속할 수 있으며, 조직 간 이동은 반드시 기존 저직에서 탈퇴 후 새 조직에 가입해야한다.**

## AWS Cost Explorer 

- AWS 비용 및 사용량을 시각화하고 분석하는 도구
- 서비스 및 계정 전반에 걸쳐 비용을 모니터링하고 예측할 수 있다.

### AWS Cost Anomaly Detection 

- AWS 계정의 비용 및 사용량 패턴을 머신러닝으로 학습하고 이상 현상이 탐지되면 즉시 알림을 보내는 고급 기능이다.
- AWS 계정 간에 비용과 사용량을 통합하여 모니터링하고 이상현상을 자동으로 감지할 수 있다.
- 임계값 설정을 통해 민감도를 조절할 수 있고, 구독 기능으로 이메일 또는 SNS를 통한 알림을 받을 수 있다.

## Cloudfront vs Global Accelerator

- 공통점   
  - 둘 다 전 세계 사용자 빠르게 연결
  - 글로벌 엣지 네트워크 사용
  - 지연시간 감소
  - AWS 글로벌 백본 네트워크 활용
  - 전 세계 사용자 대상 서비스에 적합

- 차이점
  - Cloudfront는 정적/동적 콘텐츠 캐싱
  - 웹사이트, 정적 파일 배포 등에 사용
  - Global Accelerator는 캐싱 없음 
  - 네트워크 가속 목적
  - TCP/UDP 트래픽을 가장 빠른 AWS 리전으로 라우팅
  - TCP/UDP 앱에 사용
- CloudFront는 근처 창고에서 물건 받기
- Global Accelerator는 고속도로 통해 본사까지 연결
