## 핵심 개념

- Docker는 컨테이너 기반으로 App을 실행하고 배포하는 플랫폼이다.
- `이미지, 컨테이너, 네트워크, 볼륨, 레지스트리`
- 하나하나씩 차근차근히 알아가보자.

## 컨테이너

- `애플리케이션을 실행하는데 필요한 환경과 코드, 의존성, 라이브러리 등을 포함하고 실행하는 단위다.`
- 특징
  - 빠르게 실행됨.
  - 가벼움(VM에 비해)
  - OS 커널을 공유
  - 격리된 환경을 제공한다.
- 컨테이너 1 - Spring Boot
- 컨테이너 2 - MySQL
- 컨테이너 3 - Redis 
- 각 컨테이너는 독립적으로 실행됨.

## 이미지

- `컨테이너를 생성하기 위한 템플릿`
- layer 구조로 구성되어있음.
  - 캐싱 가능
  - 빌드 속도 빠름
  - 레이어 재사용

## Dockerfile

- `Dockerfile은 이미지를 만들기 위한 설정 파일`

## Volume

- `컨테이너는 기본적으로 stateless다.`
  - 컨테이너 삭제 -> 데이터 삭제
- `컨테이너의 데이터를 유지하기 위해 Volume을 사용한다.`
- MySQL Container -> DB 데이터는 Volume에 저장

```yaml
volumes:
      - mysql_data:/var/lib/mysql
```

## Docker Network

- 컨테이너 간 통신을 담당한다.

| 네트워크    | 설명          |
| ------- | ----------- |
| bridge  | 기본 네트워크     |
| host    | 호스트 네트워크 사용 |
| overlay | 여러 서버 연결    |
