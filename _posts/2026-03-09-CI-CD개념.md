## CI/CD란?

- `CI(Continuous Integration) - 지속적 통합`
  - 코드 Push 할 때마다 자동으로 빌드 + 테스트 하는 것
- `CD(Continous Deployment - 지속적 배포`
  - CI 통과 시 자동으로 서버에 배포까지 진행하는 것
- 왜 필요한가?
  - 만약 없다면?

```
코드 수정
→ 개발자가 직접 서버 접속
→ git pull
→ docker build
→ docker run
→ 잘 됐나 확인
```

- 번거롭고 실수 발생 가능
- 있다면? 코드 푸시 -> 자동으로 다 진행


## Github Actions

- Github Actions를 사용하면 `특정 이벤트가 발생하면 자동으로 작업을 실행해준다.`
  - main 브랜치에 push -> 자동으로 빌드
  - PR -> 자동으로 테스트
  - 태그 -> 자동으로 배포
- 구성요소 
  - `Workflow`: 전체 자동화 파이프라인, `.github/workflows/` 폴더에 yaml 파일로 작성
  - `Event (on)`: 워크플로우를 언제 실행할지. push, pull_request_ schedule 등
  - `Job`: 워크플로우 안에 있는 작업 단위. 여러 Job이 병렬 또는 순서대로 실행된다.
  - `Step`: Job 안에 있는 세부 실행 단계, 명령어 하나하나가 Step이다.
  - `Runner`: Job이 실행되는 서버. Github가 무료로 제공(ubuntu, mac, windows).

```
Event (push 발생)
    ↓
Workflow 실행
    ↓
Job (build-and-push)
    ↓
Step 1: 코드 체크아웃
Step 2: Docker 빌드
Step 3: Docker Hub 푸시
```

- yaml 구조 예시

```yaml
name: CI                    # 워크플로우 이름

on:                         # 언제 실행?
  push:
    branches: [ "main" ]    # main 브랜치 push 때

jobs:                       # 어떤 작업?
  build:                    # job 이름
    runs-on: ubuntu-latest  # 어떤 Runner에서?
    
    steps:                  # 세부 단계
      - name: 코드 체크아웃
        uses: actions/checkout@v3
        
      - name: Docker 빌드
        run: docker build -t myapp .
```


## CI 실습해보기

- 프로젝트 루트에 `mkdir -p .github/workflows` 폴더 생성
- .github/workflows 내부에 `docker-build.yml` 작

```yaml
name: Docker Build and Push

on:
  push:
    branches: [ "main" ]

jobs:
  build-and-push:
    runs-on: ubuntu-latest

    steps:
      - name: 코드 체크아웃
        uses: actions/checkout@v3

      - name: Docker Hub 로그인
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}

      - name: Docker 이미지 빌드 및 푸시
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ secrets.DOCKERHUB_USERNAME }}/cicd-practice:latest
```

- github repository settings -> Secret and variables -> new repository secrets에서 DOCKERHUB_USERNAME, DOCKERHUB_TOKEN 등록
- `코드 Push`
  - git add .
  - git commit -m "first commit"
  - git push origin main
  - Actions 탭에서 워크플로우 실행 확인
- 워크플로우 성공했으면 docker hub Repositories에 Push된 것 확인 가능

## CD 실습해보기

- CD를 완성하다면 자동으로 진행되는 과정

```yaml
steps:
  - 코드 체크아웃
  - Docker Hub 로그인
  - 이미지 빌드 + 푸시
  - EC2에 SSH 접속        ← 추가
  - docker pull           ← 추가
  - docker run            ← 추가
```

결과적으로

```
내가 코드 수정 후 git push
        ↓
GitHub Actions 자동 실행
        ↓
이미지 빌드 → Docker Hub 푸시 → EC2 배포
        ↓
아무것도 안 했는데 서버에 새 버전 올라가있음
```

- `AWS에서 EC2 띄우기 (ubuntu 24.04) + t2.micro`
- `ssh 키 + Public IP로 EC2 접속`
- `EC2에 Docker 설치`
  - ubuntu에 도커 설치 시 자동으로 docker 그룹 생성 -> docker 명령어는 기본적으로 root 권한 필요하기 때문에, sudo 붙이지 않기 위해서 docker 그룹 사용자 추가
  
```
sudo apt update
sudo apt install -y docker.io
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker ubuntu
후 EC2 재접속 및 docker --version으로 설치 확인
```

![images2](/assets/images/EC2Connect.png)

- Github Actions가 EC2에 SSH 접속하려면 키 페어 필요
  - .pem파일 내용을 Github Secret에 등록
- EC2 IP도 Secrets에 등록
- docker-build.yml 파일 수정

```yaml
name: Docker Build and Push

on:
  push:
    branches: [ "main" ]

jobs:
  build-and-push:
    runs-on: ubuntu-latest

    steps:
      - name: 코드 체크아웃
        uses: actions/checkout@v3

      - name: Docker Hub 로그인
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKERHUB_USERNAME }}
          password: ${{ secrets.DOCKERHUB_TOKEN }}

      - name: Docker 이미지 빌드 및 푸시
        uses: docker/build-push-action@v5
        with:
          context: .
          push: true
          tags: ${{ secrets.DOCKERHUB_USERNAME }}/cicd-practice:latest

      - name: EC2 배포
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.EC2_HOST }}
          username: ubuntu
          key: ${{ secrets.EC2_SSH_KEY }}
          script: |
            docker pull ${{ secrets.DOCKERHUB_USERNAME }}/cicd-practice:latest
            docker stop cicd-practice || true
            docker rm cicd-practice || true
            docker run -d -p 8080:8080 --name cicd-practice ${{ secrets.DOCKERHUB_USERNAME }}/cicd-practice:latest
```

- 코드 푸시 
  - git add .
  - git commit -m "add CD pipeline"
  - git push origin main
- 보안규칙 모든 TCP 8080 추가
- publicIP:8080 접속 성공
