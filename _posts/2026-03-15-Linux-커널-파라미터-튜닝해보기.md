# Linux 커널 파라미터 튜닝 실습 정리

## 1. 실습 목적

VMware Fusion 안의 Ubuntu VM에서 Linux 커널 파라미터를 확인하고, 일부 네트워크 관련 값을 변경한 뒤, 간단한 트래픽을 발생시켜 소켓 상태를 확인한다.

이 실습의 핵심은 성능 향상을 증명하는 것이 아니라, 시스템 엔지니어 관점에서 다음 절차를 경험하는 것이다.

- 현재 설정값 확인
- 설정 변경 전 백업
- 커널 파라미터 변경
- 변경값 적용 및 검증
- 트래픽 발생
- 소켓 상태 및 시스템 리소스 확인
- 필요 시 원복

## 2. ab 부하 테스트란?

`ab`는 ApacheBench의 약자이며, Apache HTTP Server에서 제공하는 간단한 HTTP 부하 테스트 도구이다.

`ab`를 사용하면 특정 URL에 여러 번 요청을 보내고, 동시에 몇 개의 요청을 처리할지 지정할 수 있다. 이를 통해 웹서버가 요청을 처리하는 동안 연결 상태, 응답 시간, 처리량 등을 간단히 확인할 수 있다.

예시:

```bash
ab -n 1000 -c 100 http://127.0.0.1:8080/
```

옵션 의미:

```text
-n 1000  총 1000번 요청
-c 100   동시에 100개 요청
```

이 실습에서는 `ab`를 성능 측정 도구로 깊게 분석하기보다는, 커널 파라미터 변경 후 트래픽을 발생시키고 소켓 상태를 관찰하기 위한 도구로 사용한다.

## 3. 실습 환경

- Host: macOS
- Virtualization: VMware Fusion
- Guest OS: Ubuntu VM
- Test Server: Python 내장 HTTP 서버
- Load Test Tool: ApacheBench `ab`
- Socket Check Tool: `ss`, `netstat`

## 5. 현재 커널 파라미터 확인

```bash
sysctl net.core.somaxconn
sysctl net.ipv4.tcp_max_syn_backlog
sysctl net.ipv4.tcp_tw_reuse
```

명령어 설명:

```text
sysctl
Linux 커널 파라미터 값을 확인하거나 변경할 때 사용하는 명령어이다.

net.core.somaxconn
네트워크 core 영역의 somaxconn 값을 조회한다.

net.ipv4.tcp_max_syn_backlog
IPv4 TCP 연결 요청 대기 큐 값을 조회한다.

net.ipv4.tcp_tw_reuse
TIME_WAIT 소켓 재사용 설정값을 조회한다.
```

한 번에 확인:

```bash
sysctl net.core.somaxconn net.ipv4.tcp_max_syn_backlog net.ipv4.tcp_tw_reuse
```

각 파라미터 의미:

```text
net.core.somaxconn
서버가 동시에 받아들일 수 있는 연결 대기 큐의 최대값

net.ipv4.tcp_max_syn_backlog
TCP 연결 요청 중 SYN 상태로 대기할 수 있는 큐 크기

net.ipv4.tcp_tw_reuse
TIME_WAIT 상태의 소켓 재사용 허용 여부
```

주의:

`net.ipv4.tcp_tw_reuse`는 OS 버전과 커널 정책에 따라 동작 방식이 다를 수 있다. 운영 환경에서는 값을 무조건 변경하기보다, 적용 대상 시스템의 커널 버전과 서비스 특성을 먼저 확인해야 한다.

## 6. 기존 설정 백업

설정 변경 전 `/etc/sysctl.conf`를 백업한다.

```bash
sudo cp /etc/sysctl.conf /etc/sysctl.conf.bak
```

명령어 설명:

```text
cp
파일을 복사한다.

/etc/sysctl.conf
커널 파라미터를 영구적으로 설정할 때 사용하는 대표 설정 파일이다.

/etc/sysctl.conf.bak
원본 설정을 되돌릴 수 있도록 만든 백업 파일이다. .bak은 backup의 의미로 자주 사용한다.
```

백업 확인:

```bash
ls -l /etc/sysctl.conf*
```

옵션 설명:

```text
ls
파일과 디렉터리 목록을 출력한다.

-l
파일 권한, 소유자, 크기, 수정 시간 등을 자세히 출력한다.

/etc/sysctl.conf*
sysctl.conf로 시작하는 파일을 모두 조회한다. *는 여러 글자를 의미하는 와일드카드이다.
```

## 7. 커널 파라미터 설정 추가

`/etc/sysctl.conf` 파일을 연다.

```bash
sudo vi /etc/sysctl.conf
```

파일 맨 아래에 다음 내용을 추가한다.

```conf
# Network kernel parameter tuning practice
net.core.somaxconn = 1024
net.ipv4.tcp_max_syn_backlog = 2048
net.ipv4.tcp_tw_reuse = 1
```

`vi` 저장 후 종료:

```text
ESC -> :wq -> Enter
```

## 12. 소켓 상태 확인

부하 테스트 전후로 비교한다.

전체 소켓 요약:

```bash
ss -s
```

옵션 설명:

```text
ss
소켓 상태를 확인하는 명령어이다. netstat보다 최신 도구이다.

-s
전체 소켓 상태를 요약해서 보여준다.
```

TIME_WAIT 개수 확인:

```bash
ss -tan state time-wait | wc -l
```

옵션 설명:

```text
-t
TCP 소켓만 출력한다.

-a
LISTEN 상태와 연결된 상태를 포함해 전체 소켓을 출력한다.

-n
주소와 포트를 이름으로 변환하지 않고 숫자로 출력한다. 출력 속도가 빠르고 포트 확인이 명확하다.

state time-wait
TIME_WAIT 상태의 소켓만 필터링한다.

|
앞 명령어의 출력 결과를 뒤 명령어의 입력으로 넘기는 파이프이다.

wc -l
출력된 줄 수를 센다. 여기서는 TIME_WAIT 상태 소켓 개수를 확인하는 용도로 사용한다.
```

ESTABLISHED 연결 확인:

```bash
ss -tan state established | wc -l
```

옵션 설명:

```text
state established
ESTABLISHED 상태의 소켓만 필터링한다. ESTABLISHED는 TCP 연결이 수립된 상태를 의미한다.
```

LISTEN 포트 확인:

```bash
ss -tlnp
```

옵션 설명:

```text
-l
LISTEN 상태의 소켓만 출력한다. 서버가 요청을 기다리는 포트를 확인할 때 사용한다.

-p
해당 소켓을 사용하는 프로세스 정보를 함께 출력한다. 관리자 권한이 필요할 수 있다.
```

8080 포트만 확인:

```bash
ss -tlnp | grep 8080
```

명령어 설명:

```text
grep 8080
출력 결과 중 8080이 포함된 줄만 필터링한다.
```

`netstat` 사용:

```bash
netstat -tulnp | grep 8080
```

옵션 설명:

```text
netstat
네트워크 연결 상태를 확인하는 전통적인 명령어이다.

-t
TCP 소켓을 출력한다.

-u
UDP 소켓을 출력한다.

-l
LISTEN 상태의 소켓을 출력한다.

-n
주소와 포트를 숫자로 출력한다.

-p
소켓을 사용하는 프로세스 정보를 출력한다.
```

## 18. 작성 시 주의할 표현

포트폴리오나 자기소개서에는 다음처럼 쓰는 것이 안전하다.

```text
커널 파라미터 확인, 변경, 검증 절차를 실습했습니다.
```

다음 표현은 피하는 것이 좋다.

```text
커널 파라미터 변경으로 성능이 향상되었습니다.
```

실제 성능 향상을 주장하려면 변경 전후 측정값, 반복 테스트, 동일 조건 검증이 필요하다. 이번 실습에서는 성능 향상보다 시스템 설정 변경 절차와 검증 경험을 강조하는 것이 적절하다.
