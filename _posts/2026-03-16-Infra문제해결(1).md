## 상황 

운영 중인 웹 서버에서 갑자기 새로운 로그 파일이 생성되지 않고, 배포 스크립트도 실패하기 시작했습니다.<br>
서버에 접속해 디스크 상태를 확인하고, 용량이 가득 찬 원인을 찾아 해결하세요.

- 제공 데이터들

```
df -h

/dev/sda1        50G   50G     0 100% /
tmpfs           3.9G     0  3.9G   0% /dev/shm

```

- `du -sh`: 디렉토리나 파일이 실제로 얼마나 디스크를 사용하고 있는지 확인하는 명령어 [disk usage(du) summary(-s) human readable(-h)]
- `ls -lhS /var/log`: /var/log 디렉터리에 있는 로그 파일 목록을 자세히 보여주는 명령어 [자세한 정보 표시(-l) 사람이 읽기 쉽게(-h) 파일 Size 순으로 정렬(-S)]
- `lsof /var/log/app-debug.log`: 어떤 프로세스가 파일을 열고 있는지 확인하는 명령어
- `rm`: 파일은 삭제되지만 디스크 공간은 바로 안돌아옴
- `truncate`: 파일은 유지, 내용만 삭제, 디스크 공간 회복, 서비스 재시작 필요 없음
  - truncate -s [크기] 파일 -> truncate -s 0 /var/log/app-debug.log


```
lsof 출력 예시
java    1234   appuser   10w   REG   /var/log/app-debug.log
실행중인 프로그램, PID, 실제 사용자, 쓰기 모드로 파일 사용
```


## 해결 


- df -h 출력을 통해 어떤 파티션이 가득 찼는지 확인
- du 명령어로 어떤 디렉터리가 용량을 많이 차지하는지 추적해보기


```
1. 가장 큰 파일 확인하기
ls -lhS /var/log | head -5

2. 로그 비우기
truncate -s 0 /var/log/app-debug.log

3. 디스크 용량 회복 확인
df -h /

4. 앱의 로그 레벨을 INFO로 변경하기

5. logrotate 설정 추가로 재발 방지하기

cat > /etc/logrotate.d/app << 'EOF'
/var/log/app-*.log {
    daily  
    rotate 7
    compress
    missingok
    notifempty
    copytruncate
}
EOF

EOF가 나올 때 까지 내용을 /etc/logrotate.d/app 파일에 저장

daliy: 하루에 한 번 회전 (app.log, app.log / app.log.1, app.log / app.log.1 / app.log.2) 하루마다 새 로그 파일 생성 (rotate)
rotate 7: 로그를 최대 7개까지 보관
compress: 오래된 로그를 gzip으로 압축(디스크 용량 절약)
missingok: 로그 파일이 없어도 에러 발생 X (안정적으로 실행)
notifempty: 파일이 비어있으면 rotate 하지 않음
copytruncate: 일반 rotate 방식 (app.log -> 이름 바꾸기 -> app.log.1 + 새 app.log 생성)

문제 발생: 프로세스는 파일 디스크립터(fd)로 파일을 열어놓기 때문에, 파일 이름이 바뀌어도 fd는 옛날 파일(app.log.1)을 계속 가리킨다.
-> app.log에는 로그가 안쌓임

해결: copytruncate -> app.log 내용을 app.log.1로 복사 -> app.log파일을 truncate를 통해 0바이트로 비워버림 -> 프로세스의 fd는 여전히 app.log를 가리키고 있으니 trucnate 이후에도 계속 같은 파일에 로그를 작성한다.

logrotate는 실행될 때, /etc/logrotate.conf 읽고 /etc/logrotate.d 안의 파일들을 읽는다.
/app 파일은 app 프로그램 로그 관리 설정 파일로 따로 만들어주는 것
EX) /etc/logrotate.d/nginx
/etc/logrotate.d/mysql
/etc/logrotate.d/docker
/etc/logrotate.d/app

```

- `logrotate`: 로그 파일이 너무 커지지 않도록 자동으로 분리하고 압축하는 도구

- 대표적인 로그 레벨

| 레벨    | 의미         | 사용 상황      |
| ----- | ---------- | ---------- |
| TRACE | 가장 상세한 로그  | 내부 동작 추적   |
| DEBUG | 디버깅 정보     | 개발 중 문제 분석 |
| INFO  | 일반적인 실행 정보 | 정상 동작 기록   |
| WARN  | 경고         | 잠재적 문제     |
| ERROR | 오류         | 기능 실패      |
| FATAL | 치명적 오류     | 프로그램 종료 수준 |
