## Provider Block

- 각 서비스의 API 호출로 바꿔주는 플러그인을 설정하는 역할이다.

```hcl
provider "aws" {
  region     = "ap-northeast-2" # 서울 리전
  # 여기에 인증 정보를 적기도 하지만, 권장되지 않습니다 (보안!)
}
```

- 주요 설정
  - Region: 인프라가 생성될 위치
  - Credentials: access_key, secret_key 등(환경변수 or IAM Role 사용 권장)
  - Alias: 하나의 프로바이더를 여러 설정으로 쓰고 싶을 때 사용

```hcl
# 기본 프로바이더 (서울)
provider "aws" {
  region = "ap-northeast-2"
}

# 보조 프로바이더 (버지니아) - alias 사용
provider "aws" {
  alias  = "usa"
  region = "us-east-1"
}

# 리소스에서 특정 프로바이더 지정
resource "aws_instance" "seoul_server" {
  # 별도 지정 없으면 기본(서울) 사용
}

resource "aws_instance" "usa_server" {
  provider = aws.usa # 알리아스 지정!
}
```

## Resource Block

- Provider 위에 실제로 무엇을 실행할 지 결정하는 역할


```hcl
resource "aws_instance" "web_server" {
  # (1)   # (2)            # (3)
  
  ami           = "ami-0c55b159cbfafe1f0" # (4)
  instance_type = "t2.micro"              # (4)
}
```

- `resource`: 리소스 선언
- `aws_instance`: 테라폼이 어떤 종류의 자원을 만들지 정의(약속되어져 있는 이름)
- `로컬 이름(web_server)`: 이 리소스를 참조하기 위해 내가 마음대로 붙이는 별명, 실제 AWS 콘솔에 이 이름으로 뜨는 것은 아님
- `인수`: 해당 리소스를 만들기 위해 필요한 설정값들
- `물리적 이름`: 인수 안에서 `name= ...` or `tag = { Name= ...}`과 같이 지정해주어야 AWS 서비스 상에서 설정한 이름이 보이게 된다.
- `Meta Arguments`
  - 기본 설정 외에 리소스의 행동을 제어하는 기능들
  - `depends on`: 리소스 간의 생성 순서 강제 지정
  - `count`: 동일한 리소스 여러개 만들 때 사용(count = 3 이면 인스턴스 3개 생성)
  - `for_each`: 리스트나 맵 데이터를 기반으로 리소스를 반복 생성할 때 사용
  - `lifecycle`: 리소스가 삭제되거나 수정되는 방식 제어(예: prevent_destroy = true를 하면 실수로 삭제되는 것을 막을 수 있음)
- 리소스 간 참조 예시

```hcl
# 1. 보안 그룹 리소스 (문지기 역할)
resource "aws_security_group" "web_sg" {
  name        = "allow_http"
  description = "Allow HTTP inbound traffic"

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
  }
}

# 2. EC2 인스턴스 리소스 (실제 서버)
resource "aws_instance" "web_server" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = "t2.micro"

  # [중요] 보안 그룹 리소스의 ID를 참조하여 인수로 사용
  vpc_security_group_ids = [aws_security_group.web_sg.id]

  tags = {
    Name = "MyTerraformServer"
  }
}
---------------------------------------------------------------
resource "aws_instance" "app_server" {
  ami           = "ami-xyz123"
  instance_type = "t2.micro"

  # 서브넷 참조
  subnet_id     = aws_subnet.main_subnet.id

  # 보안 그룹 여러 개 참조 (리스트 형태)
  vpc_security_group_ids = [
    aws_security_group.web_sg.id,
    aws_security_group.ssh_sg.id
  ]
}
```


## Data Block

- 이미 존재하는 인프라의 정보를 조회하는 역할
- 테라폼 외부나 다른 프로젝트에서 만들어진 자원의 데이터를 읽기 전용으로 가져온다.

```hcl
data "aws_ami" "latest_amazon_linux" {
# (1)  # (2)            # (3)

  most_recent = true
  owners      = ["amazon"]

  filter {
    name   = "name"
    values = ["amzn2-ami-hvm-*-x86_64-gp2"]
  }
}
```

- `data`: 데이터 조회 선언
- `데이터 소스 타입(aws_ami)`: 어떤 종류의 정보를 찾을지 정의
- 로컬 이름(latest_amazon_linux)


- 활용 예시

```hcl
# 1. 최신 AMI 정보를 조회한다
data "aws_ami" "ubuntu" {
  most_recent = true
  filter {
    name   = "name"
    values = ["ubuntu/images/hvm-ssd/ubuntu-focal-20.04-amd64-server-*"]
  }
  owners = ["099720109477"] # Canonical
}

# 2. 조회한 정보를 리소스 생성에 사용한다
resource "aws_instance" "web" {
  ami           = data.aws_ami.ubuntu.id # 데이터 블록 참조!
  instance_type = "t3.micro"
}
```

## Variable Block

- 파라미터 또는 입력 값을 전달하는 역할
- 기본 구조

```hcl
variable "instance_type" {
  description = "EC2 인스턴스의 사양을 결정합니다."
  type        = string
  default     = "t2.micro"
  sensitive   = false # 비밀번호 같은 민감 정보일 경우 true로 설정
}
```

- 테라폼 에서 제공하는 Type

| 분류  | 타입      | 설명        |
| --- | ------- | --------- |
| 기본  | string  | 문자열       |
|     | number  | 숫자        |
|     | bool    | 참/거짓      |
| 컬렉션 | list(T) | 순서 있는 배열  |
|     | set(T)  | 중복 없는 집합  |
|     | map(T)  | key-value |
| 구조  | object  | 구조체       |
|     | tuple   | 고정 구조 리스트 |
| 기타  | any     | 아무 타입     |

- 사용 예시

```hcl
# 1. 변수 선언 (variables.tf)
variable "instance_type" {
  type    = string
  default = "t2.micro"
}

# 2. 리소스에서 사용
resource "aws_instance" "app" {
  ami           = "ami-0c55b159cbfafe1f0"
  instance_type = var.instance_type # var.변수명 으로 호출
}
```

## Output Block

- 서버의 IP나 DB의 Endpoint 등은 미리 알 수 없다. 이때 Output Block을 설정해두면 터미널 창에서 바로 해당 정보를 확인할 수 있다.

```hcl
output "instance_public_ip" {
  description = "웹 서버의 공인 IP 주소입니다."
  value       = aws_instance.web_server.public_ip
}
```

- output은 apply가 성공적으로 완료된 후에만 값을 표시한다.
- sensitive = true 를 설정하면 콘솔 로그에서 값이 숨겨지지만, State 파일에는 여전히 존재한다.


## Terraform Block

- 테라폼의 버전, 프로바이더 제약 조건, 상태 파일을 어디에 저장할지와 같은 메타 설정을 정의하는 역할

```hcl
terraform {
  # (1) 테라폼 프로그램 버전 제한
  required_version = ">= 1.5.0"

  # (2) 필요한 프로바이더 및 버전 정의
  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.0"
    }
  }

  # (3) 상태 파일 저장소 설정
  backend "s3" {
    bucket = "my-terraform-state-bucket"
    key    = "prod/terraform.tfstate"
    region = "ap-northeast-2"
  }
}
```

- backend 설정이 바뀌면 반드시 terraform init을 다시 실행해야한다.
