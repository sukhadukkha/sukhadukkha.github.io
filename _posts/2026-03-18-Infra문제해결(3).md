## 상황

## 해결

- 원인 분석
  - 핵심 에러: `Found: react@19.0.0 peer react@"^16.8.5 || ^17.0.0 || ^18.0.0" from react-beautiful-dnd@13.1.1`
  - react-beautiful-dnd@13.1.1이 React 19를 지원하지 않아서 의존성 충돌이 발생한 것.

- `그럼 로컬에서는 왜 돌아갈까?` 
  - 로컬에서는 이전에 npm install --legacy-peer-deps로 설치했거나, 기존 node_modules가 남아있어 에러가 발생하지 않았을 수 있습니다. 
  - CI는 매번 클린 설치하므로 충돌이 즉시 드러납니다.

- 전체 흐름

```
로컬 작업 (네 맥북)          GitHub (원격)           CI 서버 (ubuntu)
─────────────────────────────────────────────────────────────
1. 코드 수정
2. package.json 수정
   (react 19 추가 등)
3. npm install 실행
   → node_modules 생성
   → package-lock.json 갱신
4. git add/commit/push  →  5. GitHub에 올라감  →  6. Actions 자동 실행
                                                    → npm ci 실행
                                                    → 💥 여기서 터짐
                                                    
------------------------------------------------------------------


# 1. 로컬에서 문제 라이브러리 교체
npm uninstall react-beautiful-dnd
npm install @hello-pangea/dnd

# 2. package.json, package-lock.json이 변경됨
# 3. 코드에서 import 수정

# 4. 다시 push
git add package.json package-lock.json
git commit -m "fix: replace react-beautiful-dnd with @hello-pangea/dnd"
git push
```
```
로컬에서 교체 → push → CI가 새 package.json 보고 다시 npm ci → ✅ 성공                                                   
```

- 흐름
  - CI 서버는 매번 깨끗한 환경에서 새로 시작한다. 
  - node_modules 폴더는 .gitignore에 막혀있어서 GitHub에 올라가지도 않고, CI는 항상 package.json + package-lock.json 기준으로 설치하는 것이다.
  - 그래서 "로컬에서 고친다 = package.json을 고친다 = push한다 = CI가 새로 읽는다" 이 흐름으로 진행된다.


## package.json vs package-lock.json

- package.json
  - ^ 붙으면 "19 이상이면 아무거나" / ^ 붙으면 "1.6 이상이면 아무거나"

```json
{
  "dependencies": {
    "react": "^19.0.0",   
    "axios": "^1.6.0"     
  }
}
```

- package-lock.json

```json
{
  "node_modules/react": {
    "version": "19.0.0",        // 딱 이 버전으로 고정
    "resolved": "https://...",  // 정확히 이 URL에서 받아옴
    "integrity": "sha512-..."   // 이 파일이 맞는지 체크섬
  }
}
```

`npm install` 하는 순간 **실제로 설치된 정확한 버전**이 기록된다. 자동 생성되는 파일이다.

---

### 둘의 관계

```
npm install 실행
       ↓
package.json 읽음
"react ^19.0.0 필요하네"
       ↓
npm이 19.x 중 최신 버전 선택 → 19.0.0 설치
       ↓
package-lock.json에 "19.0.0으로 확정" 기록
```

```
npm ci 실행
       ↓
package-lock.json만 읽음
"react 19.0.0 딱 이거 설치"
       ↓
범위 같은 거 없이 lock 파일 그대로 재현
```

---

### 왜 둘 다 git에 올려야 할까?
```
package.json만 올리면
→ 팀원이 npm install 했을 때 버전이 달라질 수 있음
→ "내 로컬에선 되는데?" 상황 발생

package-lock.json도 같이 올리면
→ 모든 팀원, CI 서버가 완전히 동일한 버전 설치
→ 환경 차이로 인한 버그 없음
```
