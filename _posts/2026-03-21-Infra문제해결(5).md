## 상황

**웹 애플리케이션 서버를 시작하려 하면 즉시 에러가 발생하며 종료됩니다. 이전까지는 정상적으로 실행되었으며, 설정 변경은 없었습니다. 에러 로그를 분석하여 원인을 찾고 해결하세요.

- 제공 데이터

```
애플리케이션 시작 시도

$ ./server.sh start
Starting application server...
Error: bind(): Address already in use (port 8080)
Server failed to start. Exiting.

----------------------------------------------------

ss -tlnp | grep 8080

LISTEN  0  128  0.0.0.0:8080  0.0.0.0:*  users:(("python3",pid=4521,fd=3))

----------------------------------------------------

ps aux | grep 4521

nobody    4521  0.1  1.2  45678  12340 ?  S  Feb28  2:15  python3 -m http.server 8080
```

- `ss tlnp | grep 8080` or `lsof -i :8080`
  - ss: 소켓 상태 확인 (네트워크 상태)
  - -t: TCP
  - -l: LISTEN 상태
  - -n: 숫자로 출력
  - -p: 프로세스 정보 표시
- `ps aux | grep 4521`:
  - ps: 현재 실행 중인 프로세스 목록
  - -a: 모든 사용자
  - -u: 사용자 정보 포함
  - -x: 백그라운드 프로세스 포함

## 해결

- 8080 포트 다른 프로세스가 점유중
- ss 명령어 결과를 통해 python3 프로세스(4521)이 8080에서 LISTEN 중 확인
- ps aux 명령어를 통해 nobody 사용자가 실행한 서버 종료되지 않은 것 확인

```
1. 어떤 프로세스가 8080 포트 점유중인지 확인
ss -tlnp | grep 8080 or lsof -i :8080

2. 프로세스 종료
kill 4521

3. 프로세스 종료 확인 

ss -tlnp | grep 8080

4. 재시작

./server.sh start
```

- 팁
  - 포트 충돌은 이전 프로세스가 비정상 종료되어 포트를 반환하지 않았거나, 테스트 서버를 종료하지 않은 경우에 흔히 발생합니다. 서비스를 systemd로 관리하면 ExecStartPre에 포트 점유 확인 로직을 넣어 선제적으로 대응할 수 있습니다. SO_REUSEADDR 소켓 옵션을 애플리케이션에 적용하면 TIME_WAIT 상태의 포트도 즉시 재사용할 수 있습니다.
- `TIME_WAIT`: OS가 네트워크 연결을 안전하게 마무리 중 -> 포트는 아직 "사용 중" 처럼 보임
- `systemd`: 리눅스에서 서버(프로세스)를 실행, 관리하는 시스템
  - 서버 실행, 종료, 자동 재시작, 부팅 시 자동 실행, 로그 관리
  - `systemctl start`
  - 어떻게 추가하나? -> 서비스 파일 생성
    - `sudo vi /etc/systemd/system/my-server.service`
    - 내용 작성 -> 설정 반영 `sudo systemctl daemon-reload`

```
[Unit]
Description=My Application Server
After=network.target

[Service]
Type=simple
User=nobody
WorkingDirectory=/app

ExecStart=/app/server.sh start
ExecStop=/app/server.sh stop

# 🔥 포트 충돌 방지
ExecStartPre=/usr/bin/lsof -i :8080 && exit 1 || true

# 🔥 죽으면 자동 재시작
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

- `systemd`로 선제 대응 -> 서버 실행 전에 포트 확인 
  - `ExecStartPre=/usr/bin/lsof -i :8080 && exit 1 || true`
  - 8080 쓰고 있으면 서버 실행하지 않는다
- `SO_REUSEADDR`: TIME_WAIT 상태여도 포트 재사용 허용 (서버 코드에서 설정)
