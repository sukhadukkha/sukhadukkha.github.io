## 상황

신규 프로젝트를 위해 애플리케이션을 배포하려고 kubectl apply 명령을 실행했지만 에러가 발생하며 배포되지 않습니다. 제공된 매니페스트와 에러 메시지를 분석하여 원인을 찾고 해결하세요.

- 제공 데이터

```
kubectl apply -f app-deployment.yaml 출력

Error from server (NotFound): error when creating "app-deployment.yaml": namespaces "playground" not found

----------------------------------------------------------------------------------------------------------

app-deployment.yaml

apiVersion: apps/v1
kind: Deployment
metadata:
  name: sample-app
  namespace: playground
spec:
  replicas: 2
  selector:
    matchLabels:
      app: sample-app
  template:
    metadata:
      labels:
        app: sample-app
    spec:
      containers:
      - name: app
        image: nginx:1.24
        ports:
        - containerPort: 80
        
----------------------------------------------------------------------------------------------------------

kubectl get namespaces

NAME              STATUS   AGE
default           Active   30d
kube-system       Active   30d
kube-public       Active   30d
kube-node-lease   Active   30d        
```

- `namespace` : 쿠버네티스 안의 폴더
  - 리소스를 나누는 공간, 이름 충돌 방지, 관리/권한 분리
  - 팀 분리 : dev namespace / prod namespace
  - 서비스 분리: frontend namespace / backend namespace
  - 권한 관리: A팀 -> dev만 접근 / B팀 -> prod 접근 제한
  - 같은 namespace의 Pod: svc-api -> svc-db 바로 접근
  - 다른 namespace의 Pod: svc-api.default -> svc-db.playground (명시 필요) 
  - Pod -> Service -> Pod (svc-api 뒤에 있는 api Pod -> svc-db -> db Pod )
  - `Pod는 Service를 통해 다른 Pod와 통신한다. (Pod -> Pod 직접 통신 X)`

## 해결

- playground 라는 namespace가 존재하지 않는다.

```
1. 현재 존재하는 namespace 확인

kubectl get ns

2. playground 라는 namespace 생성

kubectl create namespace playground

3. 생성 확인

kubectl get ns playground

4. 다시 배포

kubectl apply -f app-deployment.yaml

5. playground namespace에서 생성된 Pod 상태 확인

kubectl get pods -n playground
```
