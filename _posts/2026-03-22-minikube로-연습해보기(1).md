## Pod 직접 띄워보기

```
# Pod 생성
kubectl run my-pod --image=nginx:alpine --port=80

# 상태 확인 (Running 될 때까지)
kubectl get pod my-pod -w

# Pod 내부 접속
kubectl exec -it my-pod -- sh

# 내부에서 nginx 응답 확인
wget -qO- localhost      - localhost(nginx 서버)에 요청 보내고 결과 출력
exit

# 로그 확인
kubectl logs my-pod

# Pod 상세 정보 (이벤트, IP, 노드 등)
kubectl describe pod my-pod

# Pod 정리
kubectl delete pod my-pod
```

- `-- sh`: 컨테이너 안에서 sh 실행 (ex -- pwd)
- `wget`: 웹 요청(HTTP)
- `-q(quiet)`: 로그 안보여주고 출력 깔끔하게
- `-O-`: 결과를 파일 대신 표준 출력으로 출력
- `describe`: Events 섹션 보고 트러블슈팅

## Service 실습

- Pod IP는 재시작하면 바뀐다 
- Service 는 고정된 DNS + 로드 밸런서 역할

- `NodePort`: 외부에서 클러스터 내부 Pod에 접근 가능하게 해준다.
- Node IP:30080 -> Service -> Pod:80
- Minikube 환경에서 가장 간단하게 외부 접근 방법 = NodePort

- service.yml 작성

```yaml
apiVersion: v1
kind: Service
metadata:
  name: nginx-svc
spec:
  type: NodePort                   # minikube에선 NodePort로 외부 노출
  selector:
    app: nginx                     # Deployment와 같은 라벨 — 여기서 연결됨
  ports:
  - protocol: TCP
    port: 80                       # Service 포트
    targetPort: 80                 # Pod 컨테이너 포트
    nodePort: 30080                # 외부 접근 포트 (30000~32767)
```

- service 적용

```
kubectl apply -f service.yaml

# Service 확인
kubectl get svc nginx-svc

# Endpoints 확인 (실제로 연결된 Pod IP 목록)
```

- `Service 이름 = DNS 라는 개념 확인하기`

```
# ClusterIP 타입으로 내부 통신 테스트 (Pod 내부에서)
kubectl run test-pod --image=busybox --rm -it -- sh
# Pod 내부에서:
wget -qO- http://nginx-svc   # Service 이름으로 DNS 조회
exit

-> busybox 컨테이너 하나 띄움
-> 내부로 들어가서 sh
-> nginx-svc -> 10.107.143.39 (ClusterIP)
-> Service로 요청 (10.107.143.39:80)
-> Service가 Pod들의 Endpoints 확인
-> 로드밸런싱
-> nginx 응답 반환
```
