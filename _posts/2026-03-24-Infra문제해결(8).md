## 상황

네트워크 지연이 보고되었습니다. 특정 애플리케이션이 과도한 TCP 연결을 생성하여 시스템 리소스를 소비하고 있는 것으로 의심됩니다. 활성 소켓을 분석하여 원인 프로세스를 식별하세요.

- 제공 데이터

- `ss(socket statistics) -s(summary)` 출력 결과
  - -s(summary): 요약

```
Total: 12456
TCP:   12100 (estab 11800, closed 120, orphaned 5, timewait 120)

Transport    Total     IP     IPv6
TCP          11980     11970  10
UDP          8         7      1
```


- `ss -tnp | awk '{print $6}' | sort | uniq -c | sort -rn | head -5`
  - TCP 연결 상태(ESTAB, TIME-WAIT 등)를 카운트해서 많은 순으로 출력
  - `-t`: TCP만
  - `-n`: 포트 숫자로 표시
  - `-p`: 프로세스 정보 포함
  - `awk '{print $6}'`: 6번째 열 추출
  - `sort`: 정렬 -> 동일한 프로세스끼리 묶기 위해 텍스트 순서대로 정렬
  - `uniq -c`: 중복 횟수 계산 -> 중복되는 행을 하나로 합치고, 그 옆에 연결 개수(Count) 표시
  - `sort -rn`: 역순 숫자 정렬 -r(reverse)
  - `head -5`: 상위 5개 출력

- `ss -tn state established | head -10`
  - `state established`: 연결이 확립된 ESTABLISHED 상태인 것만 골라냄

```
Recv-Q  Send-Q  Local Address:Port    Peer Address:Port
0       0       192.168.1.50:45231    203.0.113.10:80
0       0       192.168.1.50:45232    203.0.113.10:80
0       0       192.168.1.50:45233    203.0.113.11:80
0       0       192.168.1.50:45234    203.0.113.11:80

```


## 해결

- 현재 시스템에 11800개의 ESTABLISHED TCP 연결이 존재하며, crawler 프로세스가 11650개의 연결을 점유하고있다.


```
1. 활성 TCP 연결 전체 저장

ss -tnpa > /tmp/active_tcp.txt

2. 프로세스별 연결 수 확인

ss -tnp | awk '{print $6}' | grep -oP 'pid=\d+' | sort | uniq -c | sort -rn

"pid="라는 글자 찾고, 그 뒤 숫자가(\d) 한 개 이상(+) 연속으로 나오는 pid를 찾아라


3. 문제 프로세스 연결 수 제한 또는 종료 

kill 4567

```

- `grep -o(only matching) -P(perl regexp)`: Perl 방식의 강력한 정규표현식을 사용해 패턴에 딱 맞는 부분만 출력 (기본 grep보다 훨씬 복잡하고 세밀한 패턴 매칭 가능)
