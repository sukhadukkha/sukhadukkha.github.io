## ConfigMap / Secret 이란?

![img.png](/assets/images/ConfigMapAndSecret.png)

- `ConfigMap`: 민감하지 않은 설정값, DB 호스트, 포트, 로그 레벨 등등
- `Secret`: 민감한 값, 비밀번호, API 키, 토큰 등등 / base64로 인코딩되어 저장된다.
  - base64는 암호화가 아니다. 디코딩하면 바로 평문이 나온다. 실무에서는 Vault, AWS Secrets Manager랑 연동해서 사용


## ConfigMap / Secret 실습해보기

- `Step 1: ConfigMap 생성 및 적용 + 확인`
  - kubectl get configmap app-config
  - kubectl describe configmap app-config
- 

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
  namespace: default
data:
  DATABASE_HOST: "mysql-service"
  LOG_LEVEL: "info"
  APP_PORT: "8080"
```

- `Step 2: Secret 생성 및 적용 + 확인`
  - kubectl get secret app-secret
  - kubectl describe secret app-secret

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: app-secret
  namespace: default
type: Opaque
data:
  DB_PASSWORD: bXlwYXNzd29yZA==     # base64("mypassword")
  API_KEY: c3VwZXJzZWNyZXRrZXk=     # base64("supersecretkey")
```

- base64 인코딩은 직접
  - `-n`: 줄바꿈 문자(newline, \n)를 제거하라는 명령


```
# 인코딩
echo -n "mypassword" | base64
# bXlwYXNzd29yZA==

# -n 없이 실행 시
echo "mypassword" | base64
# 결과: bXlwYXNzd29yZAo= (끝에 'o='가 붙음)

# 디코딩 (확인용)
echo "bXlwYXNzd29yZA==" | base64 --decode
# mypassword
```

- `Step 3: Pod에 주입하는 2가지 방법`
  - 방법 1: 환경변수(env)로 주입
  - 방법 2: 볼륨 마운트로 주입
  - 두 가지 방법 다 사용

| 항목                | env 주입 방식           | 볼륨 마운트 접근 방식        |
|---------------------|------------------------|------------------------------|
| 환경변수            | 환경변수로 직접 주입   | 파일 형태로 접근             |
| 값 변경 시          | Pod 재시작 필요        | 자동 반영 (약간의 지연)      |
| 적합한 경우         | 단순 설정값            | 인증서, 설정 파일            |


- 환경변수로 주입

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-env
spec:
  containers:
  - name: app
    image: busybox
    command: ["sleep", "9999"]
    env:
    - name: DATABASE_HOST        # 컨테이너 안에서 쓸 변수명
      valueFrom:
        configMapKeyRef:
          name: app-config       # ConfigMap 이름
          key: DATABASE_HOST     # ConfigMap 안의 키
    - name: DB_PASSWORD
      valueFrom:
        secretKeyRef:
          name: app-secret       # Secret 이름
          key: DB_PASSWORD       # Secret 안의 키
```

```
kubectl apply -f pod-env.yaml

# 환경변수 확인
kubectl exec -it pod-env -- sh
env | grep DATABASE_HOST    # mysql-service
env | grep DB_PASSWORD      # mypassword (디코딩된 평문으로 주입됨)
exit

----------------------------------------------------------

결과 확인

k exec -it pod-env -- sh
/ # env | grep DATA
DATABASE_HOST=mysql-service
/ # env | grep DB
DB_PASSWORD=mypassword
```


- 볼륨 마운트로 주입

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-volume
spec:
  containers:
  - name: app
    image: busybox
    command: ["sleep", "9999"]
    volumeMounts:
    - name: config-volume
      mountPath: /etc/config      # 컨테이너 안에 마운트될 경로
    - name: secret-volume
      mountPath: /etc/secret
  volumes:
  - name: config-volume
    configMap:
      name: app-config            # ConfigMap을 볼륨으로
  - name: secret-volume
    secret:
      secretName: app-secret      # Secret을 볼륨으로
```


```
kubectl apply -f pod-volume.yaml

# 파일로 마운트된 것 확인
kubectl exec -it pod-volume -- sh
ls /etc/config/          # DATABASE_HOST  LOG_LEVEL  APP_PORT
cat /etc/config/LOG_LEVEL   # info
ls /etc/secret/          # DB_PASSWORD  API_KEY
cat /etc/secret/DB_PASSWORD # mypassword
exit

----------------------------------------------------------

결과 확인

k exec -it pod-volume -- sh
/ # ls /etc/config/
APP_PORT       DATABASE_HOST  LOG_LEVEL
/ # cat /etc/config/LOG_LEVEL 
info/ # 
/ # 
/ # ls /etc/secret/
API_KEY      DB_PASSWORD
/ # cat /etc/secret/DB_PASSWORD 
mypassword/ # exit
```

- `Step 4: ConfigMap / Secret 전체를 한번에 env로 주입`

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: pod-envFrom
spec:
  containers:
  - name: app
    image: busybox
    command: ["sleep", "9999"]
    envFrom:
        - configMapRef:
            name: app-config
        - secretRef:
            name: app-secret
```

```
결과 확인

k exec -it pod-envfrom -- sh 
/ # env | grep DATA
DATABASE_HOST=mysql-service
/ # env | grep DB
DB_PASSWORD=mypassword
```
