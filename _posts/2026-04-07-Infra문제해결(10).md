## 상황

배치 작업을 위해 Job을 배포했지만, Pod가 Error 상태로 반복 실패하여 Job이 완료되지 않습니다. backoffLimit이 1로 설정되어 있어 재시도 횟수도 제한됩니다. 제공된 Job 매니페스트와 상태를 분석하여 원인을 찾고 해결하세요.

- 제공 데이터
    
```
kubectl get jobs -n batch

NAME       COMPLETIONS   DURATION   AGE
fail-job   0/1           2m         2m

kubectl get pods -n batch

NAME             READY   STATUS   RESTARTS   AGE
fail-job-7x2kp   0/1     Error    0          2m
fail-job-r9m4d   0/1     Error    0          90s

fail-job.yaml

apiVersion: batch/v1
kind: Job
metadata:
  name: fail-job
  namespace: batch
spec:
  backoffLimit: 1
  template:
    spec:
      restartPolicy: Never
      containers:
      - name: worker
        image: busybox
        command: ["sh", "-c", "echo 'Processing...'; exit 1"]
        

kubectl logs fail-job-7x2kp -n batch

Processing...
```


## 해결

- Pod 로그 -> Processing...은 정상 출력
- 컨테이너 command에 exit 1이 포함되어 있어 매번 비정상 종료(Exit Code 1)된다.
- `Kubernetes는 Exit Code가 0이 아닌 경우 Job을 실패로 판단`
- `backoffLimit: 1`에 의해 1회 재시도 후 더 이상 시도 x

```
1. Job의 command 수정하여 정상 종료
exit 1 -> exit 0
command: ["sh", "-c", "echo 'Success'; exit 0"]

2. 기존 실패한 Job 삭제
kubectl delete job fail-job -n batch

3. 수정된 fail-job.yaml 적용
kubectl apply -f fail-job.yaml

4. Job 완료 확인
kubectl get jobs -n batch
```
