## 상황

현재 웹 서버에 접속하면 502 Bad Gateway 에러가 발생합니다. Nginx를 리버스 프록시로 사용하고 있으며, 백엔드 애플리케이션으로 요청을 전달하는 구조입니다. 제공된 설정 파일과 로그를 분석하여 원인을 찾고 해결하세요.

- 제공 데이터
```
nginx.conf파일

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}


------------------------------------------------------------------------------

error.log 파일

2024/01/15 10:23:45 [error] 1234#0: *1 connect() failed (111: Connection refused) while connecting to upstream, client: 192.168.1.100, server: example.com, request: "GET / HTTP/1.1", upstream: "http://127.0.0.1:3000/", host: "example.com"
2024/01/15 10:23:46 [error] 1234#0: *2 connect() failed (111: Connection refused) while connecting to upstream, client: 192.168.1.101, server: example.com, request: "GET /api/health HTTP/1.1", upstream: "http://127.0.0.1:3000/api/health", host: "example.com"

------------------------------------------------------------------------------

systemctl status app 출력

     Loaded: loaded (/etc/systemd/system/app.service.conf; enabled)
     Active: failed (Result: exit-code) since Mon 2024-01-15 10:20:00 UTC

```


## 해결

- error.log에서 Connection refused 에러 발생 -> upstream 서버에 연결할 수 없다.
- `systemctl status` 결과에서도 애플리케이션이 `EADDRINUSE: address aloread in use :::3000` 에러가 출력되고있다. 이는 포트 3000을 이미 다른 프로세스가 점유하고 있는 것이다.

```
1. 포트 3000을 점유중인 프로세스 확인

ss -tlnp | grep :3000
or lsof -i :3000

2. 해당 프로세스 종료

kill -9 <PID>

3. 애플리케이션 재시작

sudo systemctl restart app

4. 정상 실행 확인

sudo systemctl status app
curl -I http://127.0.0.1:3000/api/health
```

- 팁: EADDRINUSE 에러는 이전 프로세스가 비정상 종료되어 포트를 놓지 않았을 때 흔히 발생.
- systemd 서비스 파일에 `Restart=on-failure` 및 `RestartSec=5`를 추가하면 자동 복구가 가능하다.
- 근본적으로는 graceful shutdown(정리 다 마친 후 아전하게 종료)을 구현하여 SIGTERM 시그널을 올바르게 처리하는 것이 중요하다.


- graceful shutdown 
  - SIGTERM 받음
  - 새 요청 중단
  - 진행 중 작업 완료
  - 정리 후 종료
  - 스프링부트 코드 예시

```yaml
server:
  shutdown: graceful

spring:
  lifecycle:
    timeout-per-shutdown-phase: 30s
```

## 배운점

- 502 Bad Gateway -> upstream 문제 로그를 통해 확인해보기
- `Connection refused` + `EADDRINUSE` 연결해서 트러블슈팅
- error.log -> 연결 불가능 , systemctl status -> 포트 이미 사용중
- 원인 특정

- 근본 원인은 `graceful shutdown` 구현을 통해 제거
