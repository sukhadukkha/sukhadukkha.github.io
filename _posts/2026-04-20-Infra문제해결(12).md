## 상황

보안 감사에서 8000~9000 포트 범위에 비인가 애플리케이션이 수신 대기 중인 것이 발견되었습니다. 해당 포트를 점유 중인 프로세스를 식별하고 종료하세요.


- 제공 데이터

```
ss -tlnp | grep -E ':(8[0-9]{3}|9000)'

LISTEN  0  128  0.0.0.0:8080  0.0.0.0:*  users:(("node",pid=5678,fd=3))
LISTEN  0  128  0.0.0.0:8443  0.0.0.0:*  users:(("python3",pid=6789,fd=4))
LISTEN  0  128  0.0.0.0:8888  0.0.0.0:*  users:(("ruby",pid=7890,fd=5))
LISTEN  0  128  0.0.0.0:9000  0.0.0.0:*  users:(("java",pid=8901,fd=6))

- ss : 소켓 상태 확인 -> 어떤 포트에서 뭐가 떠있나
- : 뒤부터 8로 시작, 0~9 중 아무 숫자 3자리 OR 9000 
- 즉 :8000~8999 or :9000 필터링
------------------------------------------------------------------------------

ps aux | grep -E '(5678|6789|7890|8901)' | grep -v grep

- ps aux : a-> all users, u-> user-oriented, x-> no tty

```


## 해결

- 실무에서 비인가 판단 기준 
  - 실행 유저가 nobody
  - 실행 경로가 /tmp
  - 서비스 등록 안 된 프로세스
  - unknown 이름

```
1. 8000~9000 범위의 수신 포트 확인

2. 프로세스 종료 (kill)

3. 강제종료 필요시 (kill -9)

4. 정리된 것 확인

5. 관련 파일 제거 (rm -f /tmp/test-server.js)
```

- 정기적으로 ss -tlnp로 수신 포트 감사하여 비인가 서비스 조기 발견 필요
- firewalld or ufw로 필요한 포트만 허용하고 나머지는 차단하는 방식 적용하면 비인가 서비스 노출 방지 가능
- 감사 자동화를 위해 스크립트를 cron에 등록하는 것도 권장

```bash
#!/bin/bash
# /etc/cron.d/port-audit.sh

ALLOWED_PORTS="22 80 443"  # 허용된 포트 목록
ALERT_LOG="/var/log/port-audit.log"

# 현재 열린 포트 목록 추출
OPEN_PORTS=$(ss -tlnp | awk 'NR>1 {print $4}' | cut -d: -f2)

for PORT in $OPEN_PORTS; do
    if ! echo "$ALLOWED_PORTS" | grep -qw "$PORT"; then
        echo "[$(date)] 비인가 포트 발견: $PORT" >> $ALERT_LOG
    fi
done
```
