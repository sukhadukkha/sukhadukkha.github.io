## NSG 생성 및 인바운드 규칙 추가

- 1편에서 인스턴스를 생성하고 Reserved IP를 인스턴스에 할당해줬다. 이제는 Network Security Group 생성 및 규칙을 추가해본다.
- 외부 트래픽이 인스턴스에 들어올 수 있도록 포트를 열어준다. 
  - 22 : 서버에 원격 접속하기 위해
  - 80 : 웹서비스 제공하기 위해
  - 443 : 보안 웹서비스 제공하기 위해

- 궁금한점 : NACL에서 22번 포트를 막고, NSG에서만 22번 포트를 허용했는데도 SSH 접속이 된다. 왜그럴까?
  - AI에게 질문해본 결과, `OCI는 AWS와 동작 방식이 달라서, NACL, NSG가 AND 조건인 AWS와 달리 NACL과 NSG 중 하나만 허용되면 된다고 한다.(OR 조건)`

## 서버 초기 세팅 1 (Nginx 설치 및 접속)

- `sudo apt update && sudo apt upgrade -y` 
  - 새로 생성한 서버는 이미지 빌드 시점 이후의 보안 패치 안되있기에, 업데이트
  - 이후에 다른 패키지 설치할 때 의존성 충돌 날 수 있기에 최신 상태로 업데이트
  - apt update는 패키지 목록 갱신, apt upgrade는 실제 업그레이드
  
- Nginx 설치 및 동작 확인

  - `sudo systemctl status nginx`

- `ufw (iptables)`: OS 레벨 방화벽
  - 서버 내부에서 설정
  - 트래픽이 서버에 도달한 후 OS에서 필터링
  - Linux 커널의 iptables를 쉽게 관리하는 도구
  - 인터넷 -> NSG -> ufw -> nginx
  - `sudo apt install ufw -y`로 설치
  - `sudo ufw status` -> inactive 상태 확인
  - 그럼 왜 80포트 요청이 안되나? -> iptables 규칙 확인
  - `sudo iptables -L INPUT -n -v` -> 인바운드 트래픽만 IP 숫자로 규칙 목록 상세 출력

```bash

# sudo reboot 후 재접속해보기
```

## 라우팅 테이블 확인

```bash
ip route
```

### 출력 예시
```
default via 10.0.0.1 dev enp0s6
default via 10.0.0.1 dev enp0s6 proto dhcp src 10.0.0.10 metric 100
10.0.0.0/24 dev enp0s6 proto kernel scope link src 10.0.0.10 metric 100
10.0.0.1 dev enp0s6 proto dhcp scope link src 10.0.0.10 metric 100
169.254.0.0/16 dev enp0s6 scope link
169.254.169.254 via 10.0.0.1 dev enp0s6 proto dhcp src 10.0.0.10 metric 100
```

### 각 항목 설명

| 항목 | 설명 |
|------|------|
| `default via 10.0.0.1` | 기본 게이트웨이. 목적지 모르는 트래픽은 여기로 |
| `proto dhcp` | DHCP로 자동 할당된 라우트 |
| `metric 100` | 라우트 우선순위 (낮을수록 우선) |
| `10.0.0.0/24 scope link` | 서브넷 내부는 게이트웨이 없이 직접 통신 |
| `169.254.0.0/16` | 링크 로컬 주소. 클라우드 메타데이터 접근용 |
| `169.254.169.254` | Oracle Cloud 메타데이터 서버 주소 |

## 트래픽 흐름

```
EC2 (10.0.0.10)
    ↓
서브넷 게이트웨이 (10.0.0.1)  ← ip route에서 보이는 주소
    ↓
Internet Gateway  ← 논리적 구성요소, IP 없음
    ↓
인터넷
```

> 10.0.0.1은 IGW가 아니라 서브넷 내부 라우터다.
> Oracle이 서브넷 첫 번째 IP를 자동으로 라우터로 잡아둠.
> IGW는 콘솔에서만 확인 가능한 논리적 구성요소.

## ifconfig vs ip

| 명령어 | 상태 |
|--------|------|
| `ifconfig` | 구버전, net-tools 패키지 필요 |
| `ip addr` | 최신 표준, 기본 설치됨 |
| `netstat` | 구버전 |
| `ss` | 최신 표준, netstat 대체 |

```bash
# net-tools 설치 (ifconfig, netstat 쓰고 싶으면)
sudo apt install net-tools -y
```
