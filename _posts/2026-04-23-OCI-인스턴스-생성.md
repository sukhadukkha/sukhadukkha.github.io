## 서버 접속 원리(SSH)

- `SSH란?`
  - SSH(Secure Shell)는 네트워크를 통해 원격 서버에 안전하게 접속하기 위한 프로토콜이다.
  - 기존 Telnet은 데이터를 평문으로 전송했지만, SSH는 암호화를 통해 안전하게 통신한다.
- SSH는 `비대칭 암호화 방식 사용`
  - 공개키 (Public Key) : 서버에 등록하는 키 (`~./ssh/authrized_keys` 경로에서 확인 가능)
  - 개인키 (Private Key) : 클라이언트가 보관하는 키. 절대 노출 금지

```bash
ssh -i <개인키 경로> <유저명>@<서버IP>
```

- -i: 개인키 파일 지정
- -p: 포트 지정
- -v: 디버그 모드
- `chmod 400이 필요한 이유`
  - SSH는 보안상 개인키 파일의 권한이 너무 열려있으면 접속을 거부한다.

![img.png](/assets/images/KeyPermissionTooOpen.png)


- 22번 포트에서 포트 변경하는법

```bash
sudo vi /etc/ssh/sshd_config

# 아래 줄 수정
Port 2222  # 원하는 포트로 변경

# SSH 재시작
sudo systemctl restart sshd
```

- `SSH 에이전트에 키 등록`
  - 매번 -i 옵션으로 키 경로 지정하기 귀찮을 때, `ssh-add` 명령으로 `ssh <유저명>@<IP>` 형식으로 서버에 접속할 수 있다.

```bash
# 키 등록
ssh-add ~/.ssh/ssh-key-2026-04-22.key

# 등록된 키 목록 확인
ssh-add -l

# 키 삭제
ssh-add -d ~/.ssh/ssh-key-2026-04-22.key

# 전체 삭제
ssh-add -D
```

- Mac은 재부팅하면 ssh-agent가 초기화되서 키가 날아가기 때문에 `--apple-use-keychain` 옵션을 사용하면 키체인에 저장되어 재부팅에도 유지된다.
