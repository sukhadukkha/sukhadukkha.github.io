## 상황

시스템 리소스가 비정상적으로 큰 프로세스 트리에 의해 소비되고 있습니다. 가장 많은 자식 프로세스를 가진 부모 프로세스를 찾아 프로세스 계층 구조를 파악하세요.

- 제공 데이터

```
ps -ef | wc -l

312

ps -eo ppid --no-headers | sort | uniq -c | sort -rn | head -5

    187 159
     23 1
     12 120
      8 2
      5 345
      
pstree -p 159 | head -10

spawn_workers.sh(159)─┬─sleep(209)
                      ├─spawn_workers.sh(190)───sleep(220)
                      ├─spawn_workers.sh(191)───sleep(221)
                      ├─spawn_workers.sh(192)───sleep(222)
                      ├─spawn_workers.sh(193)───sleep(223)
                      ├─spawn_workers.sh(194)───sleep(224)
                      ├─spawn_workers.sh(195)───sleep(225)
                      ├─spawn_workers.sh(196)───sleep(226)
                      ...
                      

 ps -o pid,user,comm -p 159
 
  PID USER     COMMAND
  159 devops   spawn_workers.sh             
```


## 해결

- 전체 프로세스 개수확인 -> 범인 부모 프로세스 식별 -> 구조를 트리 형태로 파악하여 원인 특정
- 사용된 명령어 및 옵션 정리

- `ps- -ef | wc -1`
  - 전체 프로세스 상세 format으로 출력 및 몇줄인지 파악
- `ps -eo ppid --no-headers | sort | uniq -c | sort -rn | head -5`
  - 어떤 부모 프로세스가 가장 많은 자식 프로세스를 유발했는지 top5 출력
  - -e : 모든 프로세스 대상
  - -o ppid : PPID 열만 지정 출력 (-o가 열 지정)
  - --no-headers : 첫 줄에 나오는 열 이름(PPID) 텍스트 숨김
  - sort : 숫자가 섞여있으면 uniq가 제대로 작동하지 않기에, 부모 PID 번호 순서대로 정렬
  - uniq -c (count) : 연속으로 중복되는 행 합치면서, 이 번호가 몇 번 중복되었는지 출력
  - sort -rn (Reverse Numeric) : 중복 개수가 나온 결과를 바탕으로 숫자를 역순으로 정렬
  - head -5 : 상위 5개 행만 출력
- `pstree -p 159 | head -10`
  - pstree : 프로세스의 부모-자식 관계를 트리 구조의 텍스트로 출력
  - -p 159 : PID 159인 프로세스를 루트로 삼고 각 이름 옆에 PID도 출력(-p)

- PID (spawn_worker.sh)가 187개의 자식 프로세스를 생성하고있다.
- pstree에서 이 스크립트가 자신을 fork하며 sleep 프로세스를 실행하고 있다.

```
# 1. 프로세스 트리 확인 (PID와 명령어 인자 포함)
pstree -ap 159

# 2. 해당 프로세스 트리 전체 파일로 저장
pstree -ap 159 > /home/devops/process_tree_report.txt

# 3. 프로세스 트리 전체 종료 (부모와 모든 자식)
kill -- -$(ps -o pgid= -p 159 | tr -d ' ')
# 또는
pkill -P 159   # 자식 프로세스 종료
kill 159        # 부모 프로세스 종료

# 4. 정리 확인
pstree -p 159 2>/dev/null || echo "프로세스 트리 제거 완료"
```

- `pstree -ap 159`
  - -a (--arguments) : 단순히 프로세스 이름만 보여주는 것이 아닌, 실행할 때 뒤에 붙인 명령어 인자까지 보여줌.
  - -p (--show-pids) : 각 프로세스 옆에 고유 PID 표시
- `pstree -ap 159 > /home/devops/process_tree_report.txt`
  - pstree를 process_tree_report.txt로 리다이렉션
- `kill -- -${ps po pgid= -p 159 | tr -d ' ')` 
  - 프로세스 그룹 ID(PGID)를 찾아 그룹 전체 종료
  - ps -o pgid= -p 159 : 159번 프로ㅔ스가 속한 PGID 출력, 뒤에 = 붙이면 헤더 없이 숫자만 출력됨.
  - tr -d ' ' : 출력된 숫자 앞뒤에 붙어있을 수 있는 공백을 제거
  - kill -- -[PGID] : 명령어 뒤에 -를 붙이고 PGID를 넣으면(kill -- -159) 그 그룹에 속한 부모와 모든 자식 프로세스에게 종료 신호를 보낸다.

  - 대안: `pkill -P 159`, `kill 159`
  - pkill -P 159 : 부모 PID가 159번인 자식 프로세스들만 다 죽여라. (-P -> Parent PID)
  - kill 159 : 자식들 정리 후, 부모 프로세스(159) 종료
- `pstree -p 159 2>/dev/null || echo "프로세스 트리 제거 완료"`
  - 종료 명령 잘 먹혔나 확인
  - 2>/dev/null : 프로세스가 이미 죽어서 안보이면 pstree는 에러메시지 (표준에러, 2)를 뽑는다. 그 에러 메시지를 쓰레기통 (/dev/null)로 버리라는 명렁.
  - || (OR) : 앞의 명령어 실패 시 뒤의 명령어 실행
  - echo "프로세스 트리 제거 완료" : 프로세스가 성공적으로 다 지워졌기 때문에 pstree가 실패하게되고, 그 결과 하면에 메시지 출력.

- pstree -ap는 프로세스 계층을 시각적으로 보여주므로 트러블슈팅에 유용하다. 그룹 전체를 종료하려면 kill -- -[PGID] 를 사용하자.
