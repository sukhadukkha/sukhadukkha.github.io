# Linux의 루트 디렉터리의 파일들 역할 정리해보기

- 루트 디렉터리에 가서 ls 명령어를 입력해보면 많은 파일들이 있다.
- 이 파일들의 역할을 정리해보려 한다.

![img.png](/assets/images/Linux-root.png)

## 설정 및 상태 관리 디렉터리

- `etc (Editable Text Configuration)` 
  - 역할: 시스템 전체와 설치된 모든 패키지들의 텍스트 환경 설정 파일이 저장되는 곳.
  - 언제 사용할까? 
    - 웹 서버 포트나 가상 호스트를 설정할 때: `/etc/nginx/nginx.conf`
    - 서버의 부팅 디스크 (EBS, NAS 등)를 자동으로 마운트하도록 등록할 때: `/etc/fstab`
    - 서버의 DNS 네임서버 주소 변경 할 때: `/etc/resolv.conf`
    
- `var (Variable)`
  - 역할: 시스템이 돌아가면서 내용과 용량이 계속해서 동적으로 변하는 데이터가 축적되는 곳.
  - 언제 사용할까? 
    - 서버가 갑자기 안돌아서 에러 로그를 분석할 때: `/var/log/syslog` , `/var/log/nginx/error.log`
    - 패키지 매니저가 다운로드한 임시 캐시 파일을 지워 디스크 용량을 확보할 때: `'/var/cache/apt/`

- `run (Runtime Transient Data)`
    - 역할: 시스템이 부팅된 이후 지금 현재 실행 중인 서비스들의 실시간 상태 정보 담는 휘발성 메모리 공간, 재부팅하면 싹 사리지는 곳.
    - 언제 사용할까? 
      - 특정 데몬 (ex: Nginx)이 중복 실행되는 것을 막기 위해 프로세스 ID를 기록해둔 PID파일을 확인할 때: `/run/nginx.pid`
      - 프로세스 간 통신을 위한 유닉스 Socket 파일이 살아있는지 볼 때: `/run/systemd/resolve/stub-resolv.conf`


## 가상 및 장치 파일 시스템 (커널 영역)

- `proc (Process Informantion)` / `sys (System Configuration)`
  - 역할: 디스크 용량을 먹지 않는 가상 파일 시스템, 커널과 메모리, CPU의 내부 상태를 파일 인터페이스를 통해 사람이 사용할 수 있도록 맵핑해둔 곳.
  - 언제 사용할까?
    - 명령어 없이 현재 CPU 모델명이나 메모리 상세 스펙을 보고싶을 때 : `cat /proc/cpuinfo`, `cat /proc/meminfo`
    - 커널 파라미터(예: 네트워크 패킷 포워딩 기능 활성회)를 실시간으로 임시 수정할때: `echo 1 > /proc/sys/net/ipv4/ip_forward`
- `dev (Devices)`
  - 역할: 리눅스가 인식한 모든 물리적/논리적 하드웨어 장치를 파일 형태로 추상화해 모아둔 곳.
  - 언제 사용할까? 
    - 새 스토리지 볼륨을 추가한 뒤 포맷하거나 파티션을 나눌 때: `/dev/sda`, `/dev/nvme0n1`
    - 셸 스크립트에서 에러 메시지나 원치 않는 출력을 버릴때: `/dev/null`
    - 암호화 키 생성등을 위해 무작위 난수 데이터가 필요할 때

## 사용자 공간 및 마운트/외부 디렉터리

- `home / root`
  - 역할: 사용자들의 독립된 공간, 일반 사용자들은 `/home/유저명` 에 모여있고 root는 `/root`에 존재.
  - 언제 사용할까?
    - 개발자 본인의 SSH 접속 공개키를 등록해둘 때: `/home/ubuntu/.ssh/authorized_keys`
    - 내가 로그인할 때마다 자동으로 실행될 나만의 터미널 환경변수 세팅할 때: `/home/ubuntu/.bashrc`

- `usr (Unix System Resources)`
    - 역할: 시스템 초기 부팅에 필수적인 핵심 파일들을 제외하고, 사용자가 추가로 설치한 대부분의 응용 프로그램, 패키지, 라이브러리들이 들어가는 실질적인 메인 저장소.
    - 언제 사용할까?
        - 소스코드를 직접 빌드해서 전역 명령어로 설치할 때: `/usr/local/bin`
        - 특정 명령어의 매뉴얼 페이지 데이터를 확인할 때: `/usr/share/man`

- `mnt / media`
    - 역할: 외장하드, USB, 추가 블록 스토리지, NFS/NAS 같은 외부 저장 장치를 리눅스 디렉터리 트리에 연결할 때 사용하는 마운트 지점.
    - 언제 사용할까?
        - AWS에서 추가한 EBS 볼륨을 임시로 연결할 때: `mount /dev/sdb1 /mnt`
        - 회사 내부 NAS 스토리지를 서버에 연결할 때 마운트 포인트로 활용.

- `tmp (Temporary)`
    - 역할: 프로세스나 사용자가 임시 파일을 만들고 사용하는 공용 임시 공간.
    - 언제 사용할까?
        - 백엔드 애플리케이션이 사용자가 업로드한 대용량 파일을 임시로 저장할 때
        - 대용량 압축 파일을 풀기 전 중간 파일들을 잠깐 보관할 때
    - 참고:
        - 일반적으로 재부팅 시 임시 파일이 삭제될 수 있으므로 중요한 파일을 보관하면 안 된다.

## 프로그램 실행 및 라이브러리 디렉터리

- `bin -> usr/bin`
    - 역할: 일반 사용자와 관리자가 공통으로 사용하는 기본 명령어 실행 파일들이 들어있는 곳.
    - 언제 사용할까?
        - `ls`, `grep`, `awk`, `bash` 같은 기본 명령어들이 실제로 위치한 경로를 확인할 때: `which ls`

- `sbin -> usr/sbin`
    - 역할: 시스템 관리자용 명령어 실행 파일들이 들어있는 곳.
    - 언제 사용할까?
        - 네트워크 설정을 확인하거나 제어할 때: `ip`
        - 디스크 파티션을 확인하거나 수정할 때: `fdisk`
        - 서버를 재부팅할 때: `reboot`

- `lib -> usr/lib`
    - 역할: `bin`, `sbin`에 있는 실행 파일들이 동작하기 위해 필요한 공유 라이브러리 파일들이 들어있는 곳.
    - 언제 사용할까?
        - 특정 실행 파일이 어떤 라이브러리에 의존하는지 확인할 때: `ldd /bin/ls`

- `...usr-is-merged`
    - 역할: 현재 시스템이 `/bin`, `/sbin`, `/lib` 등을 `/usr` 하위 구조로 통합한 Usr-Merge 상태임을 나타내는 플래그용 디렉터리.
    - 언제 사용할까?
        - 일반 사용자가 직접 사용할 일은 거의 없다.
        - 패키지 매니저가 시스템 구조가 통합되었는지 확인하는 용도로 사용한다.

## 나머지 특수 목적 디렉터리

- `boot`
    - 역할: 리눅스 커널 파일과 부트로더 관련 파일들이 들어있는 곳.
    - 언제 사용할까?
        - 커널 업데이트 후 부팅 관련 파일을 확인할 때
        - GRUB 설정이나 멀티부팅 설정을 확인할 때

- `opt (Optional)`
    - 역할: 독립적인 구조를 가진 대형 외부 프로그램이 시스템 기본 경로와 섞이지 않도록 설치되는 공간.
    - 언제 사용할까?
        - Google Chrome, Oracle DB, Datadog Agent 같은 별도 패키지가 설치될 때
        - 특정 벤더 프로그램이 `/opt/프로그램명` 형태로 설치될 때

- `snap`
    - 역할: Ubuntu의 Snap 패키지 시스템으로 설치된 프로그램들이 격리되어 관리되는 공간.
    - 언제 사용할까?
        - Snap으로 설치한 프로그램의 파일 구조를 확인할 때
        - 예시: Docker, Nextcloud, VS Code 등이 Snap으로 설치된 경우

- `srv (Service)`
    - 역할: 서버가 외부 사용자에게 제공하는 서비스의 데이터를 저장하기 위해 만들어진 표준 디렉터리.
    - 언제 사용할까?
        - 웹, FTP, CVS 같은 서비스 데이터를 저장할 때
    - 참고:
        - 현업에서는 관습적으로 `/var/www`를 더 많이 사용하는 경우도 있다.

- `cdrom`
    - 역할: CD-ROM 장치가 마운트될 때 사용되던 경로.
    - 언제 사용할까?
        - 구형 서버나 PC에서 CD-ROM 미디어를 사용할 때
    - 참고:
        - 클라우드 서버나 최신 서버 환경에서는 대부분 비어있는 상태로 존재한다.

- `lost+found`
    - 역할: 파일 시스템 오류가 발생했을 때 `fsck`가 복구한 손상 파일 조각들을 보관하는 디렉터리.
    - 언제 사용할까?
        - 갑작스러운 정전이나 강제 종료 이후 파일 시스템 검사 결과를 확인할 때
    - 참고:
        - 정상적인 상황에서는 직접 접근할 일이 거의 없다.
