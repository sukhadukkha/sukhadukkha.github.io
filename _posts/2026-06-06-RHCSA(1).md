## 사용자 및 그룹 관리 명령어


- useradd spiderman
  - id spiderman or cat /etc/passwd | grep spiderman
- useradd -g superheros -s /bin/bash -c "Ironman Character" -m -d /home/ironman ironman
  - -g: 사용자의 기본 그룹 지정
  - -G: 사용자의 보조 그룹 지정
  - -m: 홈 디렉터리에 기본 환경 폴더 생성
  - -d: 홈 디렉터리 경로 지정
- userdel -r spiderman
- groupadd superheros
  - cat /etc/group
- usermod -G superheros,() spiderman
  - superheros,great spiderman 이런식으로 그룹 2개에도 지정 가능
  - great 그룹에서 삭제하려면 원하는 그룹만 usermod -G로 지정 / superheros spiderman
- chown <소유자>:<그룹명> <대상>
  - chown -R spiderman:superheros spiderman
- chgrp -R superheros spiderman
- passwd ironman
  - cat /etc/shadow

## /etc/login.defs 파일 및 로그인


- `PASS_MAX_DAYS` : 비밀번호 변경 없이 사용할 수 있는 최대 기간
- `PASS_MIN_DAYS` : 비밀번호 변경 후 다시 바꿀 수 없을 때까지의 최소 기간
- `PASS_MIN_LEN` : 비밀번호의 최소 길이
- `PASS_WARN_AGE` : 비밀번호 만료일 전, 사용자가 로그이 할 때 경고를 보내기 시작하는 기간
- UID / GID 할당 범위 설정
  - UID_MIN ~ UID MAX : 일반 사용자가 가질 수 있는 UID 범위 (1000~60000)
  - SYS_UID_MIN ~ SYS_UID_MAX : 시스템/데몬 계정(로그인 하지 않고 백그라운드 프로그램 돌리는 계정)이 사용할 UID 범위 (201 ~999)
  - GID 관련 설정도 동일
- CREATE_HOME : useradd 명령 시 -m 옵션을 주지 않아도 기본적으로 홈 디렉터리를 만들지 yes or no 설정
- UMASK : 새 홈 디렉터리가 생성될 때의 기본 파일 권한 마스크.
  - 077 : 소유자의 권한 0만큼 가리고, 그룹 권한을 7만큼 가리고, 타인의 권한을 7만큼 가리겠다.
  - 022 : 소유자의 권한은 0만큼, 그룹은 2만큼 -> r,x 타인도 2만큼 -> r,x
- `chage -m 5 -M 90 -W 10 -I 3  babubutt`
  - `-m 5` : 비밀번호를 바꾸면 최소 5일동안 다시 못바꿈
  - `-M 90` : 비밀번호는 최대 90일 동안 유효함
  - `-W 10` : 비밀번호 만료 10일 전부터 로그인할때마다 경고
  - `-I 3` : 비번 만료 후 3일 동안 변경하지 않으면 계정 비활성화
  - `-E 2026-12-31` : 지정한 날짜가 되면 비밀번호와 상관없이 계정 자체가 폐쇄됨
  - `chage -E -1 babubutt` : 만료일 없애는 명령어
  - `chage -E 2027-03-31 babubutt` : 만료일 덮어씌우기
- grep babubutt /etc/shadow

- createrepo 란?
  - RPM 파일들이 모여있는 평범한 폴더를 dnf가 인식할 수 있는 정식 저장소로 만들어줌.

  - dnf나 yum 같은 패키지 매니저는 파일명을 일일이 다 뒤지는게 아니라 repodata 라는 목차 파일만 읽고 작동한다.
