## 1-3. SELinux Port

비표준 포트 82에서 실행 중인 웹 서버가 서비스 제공 시 문제를 겪고 있다. 다음 조건을 충족하도록 트러블슈팅을 하라 <br>

시스템의 웹 서버가 /var/www/html에 있는 모든 기존 HTML 파일을 제공할 수 있어야 한다. (참고: 기존 파일의 내용을 삭제하거나 수정하지 말 것) <br>
웹 서버가 포트 82로 서비스가 되도록 하라<br>
웹 서버가 시스템이 시작될 때 자동으로 시작되어야 한다.<br>


- 해결


```
1. 서비스 상태 확인 (httpd) -> 죽어있는지, 실행중인지, 에러 있는지 확인

systemctl status httpd

2. 웹서버 설정 확인 -> 82포트 사용 이라고 했으므로 Listen 82 있는지 확인
없으면

grep Listen /etc/httpd/conf/httpd.conf

vi /etc/httpd/conf/httpd.conf

Listen 82로 변경

3. DocumentRoot 확인 -> 문제에서 /var/www/html의 기존 html 제공 -> 정상이면 "/var/www/html로 되어있을것.
만약 아니면 ("/web" 이런식) 수정 DocumentRoot "/var/www/html" 로 수정  

grep DocumentRoot /etc/httpd/conf/httpd.conf

4. 설정파일 수정 했다면 httpd -t 로 문법 검사 (Syntax OK 확인)

5. SELinux 상태 확인

getencforce -> Enforcing 이면 SELinux 적용 중
setenforce 0 -> Permissive 모드로 변경 -> SELinux 사용 안하고 로그만 남김

httpd가 82 포트를 사용할 수 있는지 확인 -> SELinux는 httpd -> 허용된 포트만 사용 가능

82 없으면 포트 추가

semanage port -a -t http_port_t -p tcp 82 -> httpd가 82포트 사용하게 추가

-a : add / -m : modify, -d : delete
-t : type 지정
-p : protocol 지정

설정 확인

semanage port -l | grep http_port_t

6. 방화벽 확인 -> 외부에서 접속 시 82 포트 차단 될 수 있기에 확인

firewall-cmd --list-port

방화벽 허용

firewall-cmd --permenant --add-port=82/tcp

firewall-cmd --reload 로 적용

7. 서비스 재시작 -> 설정 적용

systemctl restart httpd

8. 부팅 시 자동시작

systemctl enable httpd

9. 포트 리스닝 확인

ss -tulnp | grep 82 -> httpd 82포트 Listen 하는지 확인

10. 웹페이지 확인

curl http://localhost:82
```

```

│  │  └──── group rwx
│  └─────── owner rwx
└────────── setgid

즉
root: rwx
sysadms 그룹: rwx
others: 권한 없음

drwxrws---
      ^
      setgid
      
검증
```

## 1-8. Autofs

autofs를 설정하여 원격 사용자의 홈 디렉터리를 아래 요구대로 자동으로 마운트되독록 하세요.

- materials.example.com (172.25.254.254) 에는 NFS 공유 드렉토리 /rhome 있습니다. 이 파일 시스템에는 사용자 remoteuser1 대해 사전 구성된 홈 디렉터리가 포함됩니다.
- remoteuser1의 홈 디렉터리는 materials.example.com:/rhome/remoteuser1입니다.
- remoteuser1의 홈 디렉터리는 로컬의 /rhome 디렉토리 하위의 /rhome/remoteuser1로 자동으로 마운트 되어야 합니다. 홈 디렉터리는 사용자가 쓰기 가능해야 합니다.
- remoteuser1의 비밀번호는 flectrag입니다.

- NFS (Network File System) : 다른 서버의 디렉터리를 내 서버의 디렉터리처럼 사용하는 기술
- 문제점 : 사용 안해도 24시간 계속 연결되있어서, 사용자 100명, 서버 100대라고 가정하면 NFS 연결수가 폭증한다.
- AutoFS : 필요할때만 자동 마운트 하는 서비스

```
1. autofs 설치

rpm qa | grep autofs
dnf install -y autofs

2. 마운트 디렉터리 생성

mkdir /rhome

3. auto.master 설정

vi /etc/auto.master
추가
/rhome /etc/auto.rhome

-> AutoFS에게 /rhome 아래 규칙은 /etc/auto.rhome에서 찾으라고 알려주는 용도

4. 실제 마운트 규칙 생성

vi /etc/auto.rhome
추가
remoteuser1 -rw,sync materials.example.com:/rhome/remoteuser1

synchronous
동기식 쓰기

클라이언트
 ↓
NFS 서버 디스크에 실제 기록
 ↓
완료 응답

/rhome/remoteuser1 -> meterials.example.com:/rhome/remoteuser1 자동 연결

5. autofs 자동 시작 및 재시작 및 확인

systemctl enable --now autofs
systemctl restart autofs
systemctl status autofs

6. 실제 접근해보기

ls /rhome/remoteuser1 -> autofs는 접근해야 마운트 발생

7. 마운트 확인해보기

mount | grep rhome

8. 쓰기 권한 확인

touch /rhome/remoteuser1/testfile
ls -l /rhome/remoteuser1/testfile
```

- mkdir /rhome만 했는데 /rhome/remoteuser1은 언제생기는건가?
  - `remoteuser1  -rw,sync  materials.example.com:/rhome/remoteuser1`
  - 이 설정으로 인해 사용자가 /rhome/remoutesuer1에 접근하면 autofs가 규칙 확인 후 materials.example.com:/rhome/remoteuser1를 마운트한다.

## 1-12. 파일 권한(ACL)

Copy the file /etc/fstab to /var/tmp. Configure the permissions of /var/tmp/fstab so that:
- the file /var/tmp/fstab is owned by the root user
-  the file /var/tmp/fstab belong to the group root
-  the file /var/tmp/fstab should not be executable by anyone
-  the user "natasha" is able to read and write /var/tmp/fstab
-  the user "harry" can neither write nor read /var/tmp/fstab
-  all other users (current or future) have the ability to read /var/tmp/fstab


```
1. 파일 복사

cp /etc/fstab /var/tmp/fstab

2. 소유자, 그룹 설정, 기본 소유권 설정

chown root:root /var/tmp/fstab
chmod 644 /var/tmp/fstab

3. natasha ACL 추가

setfacl -m u:natasha:rw /var/tmp/fstab

4. harry ACL 추가

setfacl -m u:harry:--- /var/tmp/fstab

5. 확인

getfacl /var/tmp/fstab
```

## 1-18. Container - rootless, systemd 등록

- 컨테이너 이름: ascii2pdf
- 앞서 생성한 pdf 이미지를 사용하세요.
- 수동 개입 없이 시스템 재부팅 시자동으로 서비스를 시작합니다.
- 컨테이너 시작 시 /opt/file을 /incoming에, /opt/processed를 /outgoing에 자동으로 마운트하도록 서비스를 구성합니다.

```
1. 디렉터리 생성 및 권한 설정 후 walhalla로 로그인

mkdir -p /opt/file /opt/processed
chown walhalla:walhalla /opt/file /opt/processed
su - walhalla

2. 컨테이너 실행 후 마운트

podman run -d --name ascii2pdf -v /opt/file:/incoming:Z -v /opt/processed:/outgoing:Z pdf
 
-v : 볼륨마운트 -> /opt/file 을 컨테이너 내부의 /incoming 디렉터리에 마운트
:Z -> SELiux Label 자동 변경 해주는 옵션

3. 컨테이너 상태 확인

podman ps -a

4. Systemd 서비스 파일 생성

mkdir -p ~/.config/systemd/user
-> 이 경로가 사용자 서비스 파일 두는 디렉터리
cd ~/.config/systemd/user

ascii2pdf 컨테이너를 관리할 수 있는 systemd 서비스 파일 생성
podman generate systemd --name ascii2pdf --files --new

podman generate systemd : podman 컨테이너를 systemd 서비스로 변환

5. systemd 등록

ssh walhalla@node (localhost)
루트에서 loginctl enable-linger walhalla
-> 로그인하지않아도 walhalla의 user systemd를 실행하게 해주는 설정

systemctl --user daemon-reload
-> 방금만든 서비스파일 읽을 수 있도록 지시

systemctl --user enable --now container-ascii2pdf.service

6. 검증 

systemctl --user status container-ascii2pdf.service
```
