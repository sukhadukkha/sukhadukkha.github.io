## Linux 설치 및 네트워크 동작

- ISO 파일 다운받고 VM에 설치 -> 부트로더 안뜸
  - 원인 : VM 설치 시 이미지 지정해서 간편 설치되고 있었음
  - 해결 : VM 설치 시 이미지 설정하지 말고, CD/DVD에 ISO 넣어서 부팅하여 부트로더 화면 나옴
- 설치 후 ping google.com이 안먹음, cat /etc/resolv.conf 파일도 없음, 인터넷 안됨
  - 원인 : ens160이 disconnect 상태 -> DHCP 요청 X -> IP 주소 X -> Gateway X -> DNS X -> 인터넷 불가
  - 해결 : nmcli connection up ens160 (NetworkManager로 ens160 활성화) -> DHCP 요청 -> IP 주소 할당 -> Gateway 설정 -> DNS 설정 자동 수행
- `Bridge` : VM이 실제 네트워크에 직접 연결, 공유기에서 직접 IP 받음
- `DHCP (Dynamic Host Configuration Protocol)` : IP 자동으로 할당해주는 서비스
- `Bridge와 NAT의 차이` : VM이 인터넷에 나가는 방식의 차이다.
  - Bridge : VM이 공유기에 직접 연결된다. 공유기에서 직접 IP 받음, VM이 실제 PC 한대처럼 보임, 같은 공유기 안의 다른 PC 에서도 VM 접속 가능
  - NAT : VMware가 중간에서 주소 변환.
  - DHCP는 Bridge라면 공유기가 직접 IP 할당, NAT이라면 VMware가 자체 DHCP로 할당
- 검증 : Windows cmd에서 ipconfig로 네트워크 상태 확인
- 현재 상태


```
Bridge 상태

                     인터넷
                         │
                  KT 공유기
                 192.168.0.9
                         │
        ┌────────────────┴────────────────┐
        │                                 │
 Windows (Wi-Fi)                  Linux VM (Bridged)
 192.168.0.136                     192.168.0.145
```

```
NAT 상태

                     인터넷
                         │
                  KT 공유기
                 192.168.0.9
                         │
                  Windows
               192.168.0.136
                         │
                 VMware NAT
                  192.168.112.1
                         │
                  Linux VM
               192.168.112.128
```


- `SSH 차이 발생`
  - Bridge : 다른 PC에서 ssh로 접속 가능 (ssh jiho@192.168.0.145)
  - NAT : Linux는 192.168.112.x 에 있으므로 VM의 로컬 PC만 그 주소에 접근할 수 있다. 즉 다른 PC에서는 접속 불가(192.168.112.x 접속 불가 / 포트 포워딩 필요)

- 재접속 시 네트워크 설정 저장안되는 문제
  - `nmcli connection modify ens160 connection.autoconnect yes`
