# Systemd 구조와 이해 실습 2번

/home/test1.sh 스크립트는 /home/test1 디렉터리 생성

/home/test2.sh 스크립트는 /home/test1 하위에 test2 생성

systemd를 통해 2개 스크립트 각각 동작되도록 test1.service, test2.service 두개의 service unit 생성

또한 graphical 모드에서만 동작, test2.service 정지 시 생성한 디렉토리들 삭제

# Systemd Service 구성 및 장애 해결 정리

## 목표

부팅 시 다음 순서로 실행되는 서비스 구성

```text
test1.service
    ↓
test2.service
```

동작 내용

```bash
test1.sh
→ /home/test1 생성

test2.sh
→ /home/test1/test2 생성
```

---

# 1. 스크립트 생성

## test1.sh

```bash
cat << EOF > /home/test1.sh
#!/bin/sh
mkdir /home/test1
EOF

chmod +x /home/test1.sh
```

### 동작

```bash
mkdir /home/test1
```

---

## test2.sh

```bash
cat << EOF > /home/test2.sh
#!/bin/sh
mkdir /home/test1/test2
EOF

chmod +x /home/test2.sh
```

### 동작

```bash
mkdir /home/test1/test2
```

---

# 2. 서비스 생성

## test1.service

```ini
[Unit]
Description=test1
Before=test2.service

[Service]
Type=oneshot
RemainAfterExit=yes
ExecStart=/home/test1.sh
ExecStop=/bin/sh -c 'rm -rf /home/test1'

[Install]
WantedBy=graphical.target
```

---

## test2.service

```ini
[Unit]
Description=test2
After=test1.service

[Service]
Type=oneshot
RemainAfterExit=yes
ExecStart=/home/test2.sh
ExecStop=/bin/sh -c 'rm -rf /home/test1/test2'

[Install]
WantedBy=graphical.target
```

---

# 배운 점

1. `status=203/EXEC`는 실행 파일 문제를 의심한다.
2. `systemctl status` → `journalctl` → Service 파일 확인 순으로 분석한다.
3. SELinux가 `/home` 영역의 스크립트 실행을 차단할 수 있다.
4. Service용 스크립트는 `/usr/local/bin`에 두는 것이 좋다.
5. Service 파일 수정 후 반드시 `systemctl daemon-reload`를 실행한다.
6. `ExecStart` 오타는 매우 흔한 장애 원인이다.
7. 서비스 장애 분석 시 로그 확인 습관이 가장 중요하다.

# 실습 4


# Apache 8080 자동 설정 스크립트 트러블슈팅 정리

## 목표

httpd 시작 전에 스크립트를 실행하여

- Listen 8080 추가
- VirtualHost 8080 설정 추가
- index1.html 페이지 제공

---

# 스크립트

```sh
#!/bin/sh

port=$(grep "Listen 8080" /etc/httpd/conf/httpd.conf)
virtualhost=$(grep "<VirtualHost \*:8080>" /etc/httpd/conf/httpd.conf)

if [ -z "$port" ]; then
cat << EOF >> /etc/httpd/conf/httpd.conf
Listen 8080
EOF
fi

if [ -z "$virtualhost" ]; then
cat << EOF >> /etc/httpd/conf/httpd.conf
<VirtualHost *:8080>
DocumentRoot "/var/www/html"
<IfModule dir_module>
    DirectoryIndex index1.html
</IfModule>
</VirtualHost>
EOF
fi
```

---

# 문제 1 - 변수 저장 방식 오류

```sh

```

# 핵심 암기

```text
$(명령어)            → 명령 실행 결과를 변수에 저장

-z                  → 값이 비어 있으면 True

-n                  → 값이 있으면 True

Listen              → 포트 오픈

VirtualHost         → 포트별 요청 처리

DocumentRoot        → 웹 문서 루트

DirectoryIndex      → 기본 페이지 설정

203/EXEC            → 실행 실패

Error 2             → 파일 없음

Error 13            → 권한 문제 또는 SELinux

localhost 접속 실패 → Apache 설정 문제

외부 접속 실패      → 방화벽 확인

8080 사용 시
firewall-cmd --add-port=8080/tcp 필요
```
