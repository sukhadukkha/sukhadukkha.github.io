## 리눅스 배포판들 종류

- `RedHat EnterPrise Linux` : 기업 대부분이 RHEL 사용, 이유는? -> 레드햇의 유지보수를 받을 수 있음 + 가장 검증 되어있는 리눅스 배포판
  - Fedora 프로젝트를 RedHat이 운영 -> Fedora에 더해 레드햇 개발자들이 더 검증해서 출시하는 것이 RHEL이다.
- `CentOS` : 본래 무료 및 공식적으로 유지보수까지 지원하였으나 몇년 전 발표로, 더 이상 공식적으로 지원하지 않는다는 소식에 RedHat으로 대부분 마이그레이션.
- `Fedora` : RedHat이 후원하는 오픈소스 커뮤니티 기반의 리눅스 배포판, 새로운 기술과 기능을 가장 먼저 도입하는 테스트베드 역할.
  - 최신 커널과 기능을 접하기에는 좋지만, 안정성이 조금 떨어진다.
- `Rocky Linux` : CentOS 창립자인 그레고리 커처가 CentOS 초심 되찾기 위해 만든 배포판.
- Debian 계열의 Ubuntu : 가벼운 Ubuntu 계열도 존재.

# 2장. 명령줄 액세스  

## Bash Shell

- 명령줄은 컴퓨터 시스템에 명령을 입력하는 데 사용되는 CLI다. Linux 명령은 쉘이라는 프로그램에서 제공된다.
- `GNU Bourne-Again Shell (bash)` : bash 쉘은 UNIX 시스템에서 원래 Bourne Shell(sh)의 개선된 버전.
- 쉘 프롬프트에서 입력된 명령은 실행할 명령, 명령의 동작 조정을 위한 옵션, 명령의 타겟이 되는 인수로 이루어져 있다.
- `Shell은 왜 필요하고 어떻게 동작할까?`
  - Kernel은 0과 1로 이루어진 기계어만 이해할 수 있다. 하지만 우리는 명령줄에 cd, mkdir 같은 문자 명령을 사용한다.
  - 이 사이에서 사용자가 입력한 문자를 기계어로 번역해서 커널에 전달하고, 커널의 처리 결과를 인간의 언어로 바꾸어 화면에 보여주는 번역기 역할을 하는 것이 Shell이다.
  - Shell 이라는 이름이 붙은 이유도 커널을 감싸고 있는 껍데기 라는 뜻이다.

# 3장. 명령줄에서 파일 관리

## Linux 파일 시스템 계층 구조


![img.png](/assets/images/Directory.png)

- / 디렉터리 -> 최상위 디렉터리

| 디렉터리 | 목적 |
|----------|------|
| `/boot` | 부팅 프로세스를 시작하기 위한 파일이 저장되는 디렉터리입니다. |
| `/dev` | 시스템에서 하드웨어에 액세스하는 데 사용하는 특수 장치 파일(Device Files)이 저장됩니다. |
| `/etc` | 시스템별 구성(Configuration) 파일이 저장되는 디렉터리입니다. |
| `/home` | 일반 사용자의 홈 디렉터리입니다. 사용자 데이터와 개인 설정 파일을 저장합니다. |
| `/root` | 관리자(root) 계정의 홈 디렉터리입니다. |
| `/run` | 마지막 부팅 이후 시작된 프로세스의 런타임 데이터가 저장됩니다. 프로세스 ID(PID) 파일, 잠금(Lock) 파일 등이 포함되며 재부팅 시 내용이 다시 생성됩니다. 이전의 `/var/run`, `/var/lock`을 통합한 디렉터리입니다. |
| `/tmp` | 임시 파일을 저장하는 디렉터리입니다. **10일 동안** 접근·수정되지 않은 파일은 자동으로 삭제됩니다. `/var/tmp`도 임시 디렉터리지만 **30일 동안** 접근·수정되지 않은 파일이 삭제됩니다. |
| `/usr` | 설치된 소프트웨어, 공유 라이브러리, 헤더 파일, 읽기 전용 프로그램 데이터가 저장됩니다. 주요 하위 디렉터리는 다음과 같습니다.<br><br>• `/usr/bin` : 일반 사용자 명령어<br>• `/usr/sbin` : 시스템 관리 명령어<br>• `/usr/local` : 로컬 사용자 설치 소프트웨어 |
| `/var` | 재부팅 후에도 유지되는 가변(Variable) 데이터가 저장됩니다. 로그 파일, 데이터베이스, 캐시, 프린터 스풀, 웹 사이트 콘텐츠 등이 위치합니다. |


- `/boot` : 서버에 전원을 켜먼 가장 먼저 이 디렉터리 찾아간다. 커널과 부팅을 도와주는 부트로더(GRUB) 파일이 위치해있다.
- `/dev (Devices)` : 리눅스는 모든 장치를 파일로 취급한다는 철학이 있다.
  - 마우스, 키보드, CPU, 디스크 드라이브 등 모두 이 디렉터리 안에 파일 형태로 존재한다.
  - Dell 서버에 새로운 디스크 하드웨어를 장착하거나 스토리지를 연결하면, 이 디렉터리 안에 sda,sdb or nvme0n1 같은 파일이 새로 생겨난다.
- `/etc (Editable Text Configuration)` : 모든 시스템 설정 파일이 모여있다.
  - 리눅스의 모든 환경설정을 바꿀때 이 디렉터리 및에 있는 파일들을 수정하게된다. 서버 백업 시 가장 중요하게 1순위로 복사해두어야 하는 디렉터리다.
- `/home/ vs /root` 
  - 왜 분리했을까? -> 리눅스는 다중 사용자 시스템이다. 일반 사용자들이 막 사용하는 공간 및 시스템 전체를 통제하는 root 공간을 엄격히 분리하기 위함이다.
- `/run` : 서버가 켜져있는 동안 프로세스 들이 사용하는 임시 메모리 공간.
- `/tmp vs /var/tmp` 
  - 왜 주기가 다를까? -> /tmp 는 휴지통 같은 곳 (주로 메모리에 생성되어 재부팅하면 다 날아간다)
  - 반면 /var/tmp는 디스크(저장장치)에 생성되므로 용량이 더 크고, 재부팅해도 유지되며 30일간 보관된다.
  - 대용량 백업이나, DB 마이그레이션 작업을 할 때, 임시 파일 공간이 필요하다면, 금방 지워지고 용량이 작은 /tmp 대신 /var/tmp를 사용해야 안전하다. 
- `/usr (Unix System Resources)`
  - /usr/bin -> 일반 사용자가 쓰는 명령어들(ls, cd, mkdir)이 있다.
  - /usr/sbin -> SystemBinaries 의 약자로 root만 사용할 수 있는 강력한 명령어들이 모여있다.
  - /usr/local -> 리눅스 패키지 매니저가 아니라, 엔지니어가 소스코드를 직접 다운받아 수동으로 설치한 프로그램들이 저장되는 곳이다.
- `/var (Variable)` : 프로그램이 실행되면서 계속 크기가 변하는 파일들이 위차한다.
