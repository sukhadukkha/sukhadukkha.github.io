/dev (device) : 하드웨어 장치 파일들<br>
/etc : 각종 Service 설정 파일들, 각종 소프트웨어들<br> 설정파일
/home : 각각의 사용자 디렉터리<br>
/ : 최상위 디렉터리<br>
/root : 관리자 디렉터리<br>
/run : 서비스 실행중일 때 임시로 생성되는 파일들, 실시간 업데이트 정보값들<br>
특정 PID 정보 확인 가능<br> 
/sbin : root만이 사용할 수 있는 binary 디렉터리<br>
/tmp : sticky 비트 적용되어져서 만든 사용자만 실행 가능, 루트는 예외<br>
/usr : 리눅스 용 소프트웨어들 대부분 /usr 밑에 설치됨<br>
/var : 실시간 데이터 동적 변경, 가벼운 로그 같은것들<br>

# 07/06

1. 절대경로 상대경로 말로 할 수 있을정도
2. rm -rfi 중 f가 우선임
3. rm으로 한번에 지울때는 rm -rf (조심해야함)
4. 링크는 보통 심볼릭링크 사용 ln -s
5. 하드링크는 원본파일(A)가 지워져도 링크파일 B는 살아남음, 디렉터리는 안되고 파일만 가능
6. 소프트링크는 B는 쓸모없어짐
7. inode -> 특정 파일의 정보를 가지고 있음. (상태, 타입, 위치, 그룹, 소유권, 권한 등등) , 파일 식별
8. 파일시스템 포맷 별로 -> 파일시스템 크기에 의해서 inode 크기 제한 (inode 꽉찰수있음)
9. 하드링크는 동일한 파일시스템일때만 작동함
10. 심볼릭링크는 파일시스템 달라도 연결 가능
11. 특정 파일 지울때는 심볼릭링크도 같이 지워야함 ( 나중에 그 심볼릭 링크 다시 연결될수있음 , 동일한 파일명 생성시)

ln -s [원본(목적지)] [링크이름]

링크이름이 이미 디렉터리이면
그 디렉터리 안에 링크 생성

12. *, ? , [abc...] , [!abc...] 는 가끔 사용, 눈에 익혀놓자
13. 대괄호 중괄호 소괄호 구분(ShellScript 작성 시 사용하면 편함)
14. 항상 변수표현시에 중괄호 써줘라
15. a=date , a=$(date), a=`date`(백틱) '' 와 헷갈릴 수 있으니 가능하면 $사용
16. a=$(date) echo '$a' , echo "$a", echo $a 의 차이 \$a 는 문자그대로 표시

17. 파일시스템은 저장장치에 파일을 저장하고 관리하는 규칙이다.

파일 이름, 파일 위치, 파일 크기, 디렉터리 구조, 권한 등의 정보를 관리한다.

SSD나 HDD는 단순히 데이터를 저장하는 공간일 뿐 파일을 직접 관리하지 못한다. 따라서 파일의 위치와 정보를 체계적으로 관리하기 위한 규칙이 필요한데, 이를 파일시스템이라고 한다.

18. ext4 : 안정성, 성능 좋은 기본 파일시스템
19. xfs : RHEL 계열 기본, 대용량 파일 처리 우수, 서버 환경에서 많이 사용
20. ''는 문자 그대로, ""는 변수값 출력
21. /dev/null 의 역할 (출력 값 버리는 곳)
22. 파이프라인 -> 이전의 명령에서 출력되는 값을 파이프라인 이후에 사용하는 옵션의 인자값으로 넘겨줌
23. 리다이렉션 & 의미
